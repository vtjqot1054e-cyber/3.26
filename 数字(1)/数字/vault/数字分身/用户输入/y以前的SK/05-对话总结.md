# 对话总结 Skill

> 职责：**实时记录**对话中的关键信息，确认后存入知识库

---

## 核心机制：实时记录 + 确认存入

### 工作模式

```
对话进行中
    ↓
AI内部实时标记关键点（用户不可见）
    ↓
触发确认（结束/轮数/手动）
    ↓
AI输出关键点清单，让用户勾选
    ↓
用户确认后，AI执行存入
```

### 实时标记的内容类型

| 类型 | 示例 |
|------|------|
| **决策/结论** | "新公司不做行政类业务" |
| **新认知/洞察** | "关系资产托管是核心风险" |
| **画像补充/修正** | "超级代码→超级代价" |
| **人物新信息** | "成的边界需要调整" |
| **待办/下一步** | "需要和成正式谈" |
| **重要事件** | "上海送礼被拒" |

---

## 触发时机

### 1. 对话结束时（必触发）
当你说以下任何一句时触发：
- "总结一下"
- "今天就到这"
- "先这样"
- "结束"
- "存一下"

### 2. 重要内容检测（即时提醒）
检测到以下情况时，AI会问"这个要记下来吗？"：
- 明确的决策/结论
- 对之前认知的修正
- 新的业务方向/战略调整

### 3. 手动触发
- "存一下" / "记一下" / "检查关键点"

> **注意**：全量对话记录已由外部监控工具（`brain/对话监控.pyw`）自动完成，本SK只需提取关键点。

---

## 确认流程

### 输出格式

```
本次对话关键点：

1. ☐ [决策] 新公司不介入行政类业务
2. ☐ [修正] 核心画像：超级代码→超级代价
3. ☐ [洞察] 关系资产托管在销售手里是核心风险
4. ☐ [人物] 成的边界需要重新定义
5. ☐ [待办] 需要和成正式谈利益分配

---
确认存入哪些？回复数字（如"1,3,5"）或"全部"
不存任何内容回复"跳过"
```

### 用户确认后

```
已存入：
- ✅ 核心画像.md：修正"超级代码→超级代价"
- ✅ 动态状态.md：新增待办"和成正式谈"
- ✅ 事件库/2026-01-25-xxx.md：新建事件

未存入（下次可补）：
- 1. 新公司不介入行政类业务
- 4. 成的边界需要重新定义
```

## 存入规则

| 内容类型 | 存入位置 |
|----------|----------|
| 决策/结论 | 相关事件文件 或 动态状态 |

## 单次对话总结（完整版）

### 提取维度

| 维度 | 问题 |
|------|------|
| **事件** | 本次对话讨论了什么事？ |
| **决策** | 做了什么决定？或面临什么选择？ |
| **情绪** | 表达了什么情绪？ |
| **洞察** | 有什么新的自我认知？ |
| **行动** | 下一步要做什么？ |
| **人物** | 提到了哪些人？有什么新信息？ |
| **模式验证** | 是否印证了核心画像中的某个模式？ |
| **模式偏离** | 是否有与核心画像不一致的表现？ |

### 输出格式

```markdown
## 对话总结 - [日期] [主题]

### 关键事件
-

### 决策/选择
- 决定：
- 理由：

### 情绪状态
-

### 新洞察
-

### 待办行动
- [ ]

### 人物更新
- [[人名]]：新信息

### 画像验证
- ✅ 印证：[哪个模式]
- ⚠️ 偏离：[哪个模式] - 需要关注

### 知识库更新建议
- [ ] 更新 [[核心画像]]：
- [ ] 更新 [[动态状态]]：
- [ ] 新建事件：[[事件名]]
```
