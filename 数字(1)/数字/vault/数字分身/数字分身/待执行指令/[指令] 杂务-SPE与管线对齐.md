# 指令：SPE文件与管线目录对齐

## 角色定位

你是杂务窗口。机械执行，产出对照表，不做价值判断。

## 任务描述

盘点 `.specstory/history/` 下所有SPE文件，与管线四个目录（待确认/对话导出/对话存档/对话删除）做交叉比对，产出一张完整对照表。

## 具体步骤

### 第一步：收集所有SPE文件

列出 `D:\数字分身2\.specstory\history\` 下所有 `.md` 文件。
提取每个文件的**时间戳**部分（如 `2026-03-20_06-38-11Z`）和**标题**部分。

### 第二步：收集管线四目录的文件

分别列出以下四个目录的所有 `.md` 文件：
- `vault\数字分身\待确认\` — 前缀 `[待确认]`
- `vault\数字分身\对话导出\` — 前缀 `[导出]`
- `vault\数字分身\对话存档\` — 前缀 `[对话]` 或 `[对话存档]`
- `vault\数字分身\对话删除\` — 前缀 `[对话删除]`

提取每个文件的**时间戳**部分。

### 第三步：交叉比对

用时间戳做匹配键（SPE文件名里的时间戳 vs 管线文件名里的时间戳）。

对每个SPE文件，判断它在管线中的状态：
- **待确认** — 在待确认目录中找到匹配
- **已存档** — 在对话存档目录中找到匹配
- **已删除** — 在对话删除目录中找到匹配
- **仅导出** — 在对话导出目录中找到匹配，但没进待确认
- **未进管线** — 四个目录都没找到匹配

注意：
- 时间戳可能有秒版本（`2026-03-20_06-38-11Z`）和分钟版本（`2026-03-20_06-38Z`），匹配时**去掉秒**做模糊匹配（`2026-03-20_06-38` 能匹配 `2026-03-20_06-38-11Z`）
- 一个SPE可能同时在多个目录有对应文件（比如导出+待确认），标注最终状态

### 第四步：检查待确认文件的审阅状态

对所有在"待确认"状态的文件，读取frontmatter中的 `status` 字段：
- `pending` — 未审阅
- `reviewed` — 已审阅（用户写了引子）
- `done` — 已完成（复盘大脑处理过）
- `cancelled` — 已取消

### 第五步：产出对照表

写入文件：`vault\数字分身\AI产出\系统搭建\[产出] SPE与管线对齐表-2026-03-23.md`

格式：

```markdown
# SPE与管线对齐表

> 生成时间：2026-03-23
> SPE总数：XX个
> 待确认：XX个（pending XX / reviewed XX / done XX / cancelled XX）
> 已存档：XX个
> 已删除：XX个
> 仅导出：XX个
> 未进管线：XX个

## 对照表

| 序号 | SPE文件名 | 时间戳 | 管线状态 | 管线文件名 | 审阅状态 |
|------|----------|--------|---------|-----------|---------|
| 1 | xxx.md | 2026-03-20_06-38-11Z | 待确认 | [待确认] xxx.md | pending |
| 2 | xxx.md | 2026-03-20_07-33-08Z | 未进管线 | — | — |
| ... | | | | | |

## 未进管线的文件清单

| 序号 | SPE文件名 | 时间戳 | 备注 |
|------|----------|--------|------|
| 1 | xxx.md | ... | |

## 今日新增SPE（2026-03-23）

| SPE文件名 | 管线状态 |
|----------|---------|
| ... | ... |
```

### 第六步：报告

完成后告诉用户：
- 对照表路径
- 未进管线的文件数量
- 今日新增SPE数量
- 是否有异常（如同一时间戳在多个目录出现）

## 注意事项

- **只产出对照表，不移动/创建/删除任何文件**
- 如果SPE文件数量很多（>100），先数数报告总量，再分批处理
- 时间戳匹配要考虑秒/分钟两种格式的兼容
- 对照表按时间戳倒序排列（最新的在前）
