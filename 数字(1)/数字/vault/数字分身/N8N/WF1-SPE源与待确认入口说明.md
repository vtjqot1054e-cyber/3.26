# WF1：SPE 源、总仓库根、待确认入口（2026-03-21）

## 事实：本机存在三处 SpecStory history

| 位置 | 典型场景 |
|------|----------|
| `E:\----2\.specstory\history\` | Cursor **工作区开在仓库根** `E:\----2` 时，SpecStory 写这里（「总」） |
| `vault\数字分身\.specstory\history\` | 工作区开在「数字分身」子项目时（分1） |
| `vault\数字真我\.specstory\history\` | 工作区开在「数字真我」子项目时（分2） |

## 事实：当前 WF1（`01-A对话清洗标注.json`）只扫其中两处

- `speDir1` = `vaultPath` + `.specstory/history` → **数字分身**
- `speDir2` = `vaultPath/../数字真我/.specstory/history` → **数字真我**
- **未包含** `E:\----2\.specstory\history`（仓库根）

因此：若你主要在 **总仓库根** 窗口里干活，新 SPE 落在 **根目录 `.specstory`**，WF1 **不会处理**，表现为「跑不通 / 没新文件」——**不是逻辑错误，是管线未覆盖第三源**。

## 事实：待确认（管线产出）只有一处

WF1 **写入待确认** 的路径在 **数字分身** 下：`vault/数字分身/待确认/`（与 `vaultPath` 一致）。

- **数字真我** 项目内 **没有** 与之平行的第二条「待确认」产出目录（除非另做工作流）。
- **OB**：只需打开 **一个 vault 根**：`E:\----2\vault`（或你习惯的 vault 根），在侧栏进入 **`数字分身/待确认`** 即可审阅；**不必**「分身、真我各开一个库专门找待确认」。
- **IDE**：两个 Cursor 窗口对应两个 SPE 落盘位置，切换确实烦；**审阅对象**仍是 **同一文件夹** `数字分身/待确认`，与开几个窗口无关——烦的是 **SPE 分散**，不是待确认分散。

## 推断（与你的「总-分」说法对齐）

- 「要一个总的 SPE」若指 **自动汇总** 到一处：SpecStory **不会**帮你合并，只能靠 **WF1 多源扫描** 或 **符号链接/同步脚本**（工程决策）。
- 「WF1 分别提取」：当前是 **两源**；要覆盖 **总仓库根 SPE**，需在 **SpecStory预清洗** 节点增加 **第三目录** `E:\----2\.specstory\history`（或相对 `vaultPath` 的 `../../.specstory/history`），并注意 **同名文件** 在两处时的去重顺序。
