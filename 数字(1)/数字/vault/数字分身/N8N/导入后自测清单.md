# N8N 导入后自测清单（WF1 / WF3）

> 本机 n8n 须用 **`E:\n8n\start-n8n.ps1`** 启动（含 `NODE_FUNCTION_ALLOW_BUILTIN=child_process,fs,path`）。

## 导入

1. Workflows → **Import from File**
2. 依次导入（路径以仓库为准）：
   - `vault/数字分身/N8N/01-A对话清洗标注.json`
   - `vault/数字分身/N8N/02-用户校对提炼.json`（若不用可跳过）
   - `vault/数字分身/N8N/03-用户校对归档.json`

## WF1 快速测

1. 在 **`vault/数字分身/对话导出/`** 放入或确认有测试用 `.md`（或依赖 SpecStory 双源扫描）。
2. 打开 **01-对话清洗标注** → 点 **手动触发**（或等定时）。
3. **SpecStory预清洗** 应不再报 `child_process is disallowed`。
4. 检查 **`vault/数字分身/待确认/`** 是否生成/更新预期文件。

## WF3 快速测（可选）

1. 将某篇 `待确认` 文件 frontmatter 中 **`status` 改为 `done`**（或 `cancelled` 测删除流）。
2. 运行 **03-用户校对归档** 工作流。
3. 检查 **`对话存档/`** 或 **`对话删除/`** 是否出现对应归档文件。

## 异常

| 现象 | 处理 |
|------|------|
| `child_process` disallowed | 用 `start-n8n.ps1` 重启 n8n，勿用手动 `npx` 且漏设环境变量 |
| Python 相关 | 工作流里清洗脚本使用 **`py`** 调用（Windows 下 `python` 常为商店占位符）；需本机已装 Python Launcher |
