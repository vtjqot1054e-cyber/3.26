### 🛠️ n8n 前置黑话翻译器 (Pre-Translator) 部署蓝图

#### 一、 核心功能与终极目标

* **功能定位：** 整个 1+N+1 数字系统的“第一道物理净水器”。
* **目标：** 将充满口误、错别字、随机代号的原始语音转写文本，进行**0 成本、0 幻觉的强制降噪**。确保后续进入主控大脑（Claude）和 SK-04 碎片整合节点的数据，是 100% 具备标准实体名称（Entity）的干净事实，从而保证 Obsidian 知识库双链的绝对精准。
* **红线原则：** 本环节**绝对禁止**调用大模型进行语义猜测。只做确定性的本地字典正则替换。

#### 二、 n8n 工作流架构设计 (Workflow Nodes)

在 n8n 的可视化画布中，只需串联以下 4 个节点：

1. **Node 1: Webhook (触发器)**
* **作用：** 暴露出一个本地的 URL 接口。你通过任何顺手的快捷工具（如 Mac 快捷指令、Python 小弹窗），将剪贴板里的烂文本 POST 到这个地址。
* **输入变量：** `raw_text` (原始文本)

2. **Node 2: Read File (读取本地资产)**
* **作用：** 每次触发时，瞬间读取本地硬盘上的 `aliases.json` 文件。
* **输出变量：** `dictionary_data` (字典 JSON 对象)

3. **Node 3: Code (核心洗稿引擎)**
* **作用：** 运行一段极简的 JavaScript 脚本。它将字典中的 Key 提取出来，**强制按字符串长度降序排列**（极度重要：防止短词误杀长词，例如防止把“新会务公司”错误替换成“新[某某]公司”），然后对 `raw_text` 进行全局正则替换。

4. **Node 4: Execute Command / HTTP Request (交接发包)**
* **作用：** 把洗干净的标准文本，作为参数直接传给本地终端里待命的 Claude 脚本（触发你的 SK-04 碎片整合），或者返回给你的前端双屏比对 UI。

#### 三、 核心资产结构：`aliases.json`

要求该字典文件结构极简，直接采用键值对平铺，方便后续脚本一键追加。

```json
{
  "兰心": "运营B",
  "卖壳的": "A公司",
  "正委": "政委工具",
  "聋瞎": "OpenClaw",
  "小聋瞎": "OpenClaw",
  "A流": "文字流"
}

```

#### 四、 核心节点代码 (提供给 Node 3 的 JavaScript)

请 Claude 审查以下将在 n8n Code 节点中执行的核心清洗逻辑：

```javascript
// 1. 抓取前置节点数据 (需根据实际 n8n 节点命名微调)
const rawText = $('Webhook').first().json.body.raw_text; 
const dict = $('Read File').first().json; // 假设读取到的 JSON 直接映射为对象

let cleanText = rawText;

// 2. 核心容错：提取所有的词根，并按长度从长到短排序
// 目的：先替换长词，再替换短词，防止发生词根重叠导致的误杀
const keys = Object.keys(dict).sort((a, b) => b.length - a.length);

// 3. 暴力洗稿：执行全局正则替换
keys.forEach(key => {
  // 加上 'g' 标签确保替换整段文本中的所有匹配项
  // 如果遇到特殊正则字符，可在此处加入转义逻辑（当前为极简版）
  const regex = new RegExp(key, 'g'); 
  cleanText = cleanText.replace(regex, dict[key]);
});

// 4. 组装标准输出结构
return {
  json: {
    status: "success",
    original_text: rawText,
    clean_text: cleanText,
    timestamp: new Date().toISOString()
  }
};

```
