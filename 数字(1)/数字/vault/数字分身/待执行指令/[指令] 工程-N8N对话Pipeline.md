## 工作流1：对话清洗标注

### 流程图

```
触发（Schedule 30min + Manual）
  │
  ├─→ 读取实体字典（GitHub: 数字分身/全局实体字典.md）
  │
  ├─→ 列出对话导出目录（GitHub: 数字分身/对话导出/）
  │
  ├─→ 列出待确认目录（GitHub: 数字分身/待确认/）
  │
  └─→ 过滤未处理（对话导出有、待确认没有的文件）
        │
        ▼
      逐个读取原文（GitHub）
        │
        ▼
      清洗+字典替换+标记（Code节点）
        │
        ▼
      写入待确认/（GitHub）
```

### 节点详细规格

#### 节点1：触发
- 类型：`n8n-nodes-base.scheduleTrigger`（每30分钟）
- 同时保留一个 `n8n-nodes-base.manualTrigger` 作为手动入口
- 两个触发都连到节点2

#### 节点2：读取实体字典
- 类型：`n8n-nodes-base.github`
- 操作：file → get
- owner: `vtjqot1054e-cyber`
- repo: `----2`
- 路径：`数字分身/全局实体字典.md`
- 凭据占位：`githubApi`（用户导入后自行绑定）
- **注意**：GitHub API 返回的 content 是 base64 编码，必须解码

#### 节点3：列出对话导出目录
- 类型：`n8n-nodes-base.github`
- 操作：file → list
- 路径：`数字分身/对话导出`

#### 节点4：列出待确认目录
- 类型：`n8n-nodes-base.github`
- 操作：file → list
- 路径：`数字分身/待确认`

#### 节点5：过滤未处理
- 类型：`n8n-nodes-base.code`
- 模式：`runOnceForAllItems`
- 逻辑：
  - 取节点3的文件列表（只要 `.md` 文件）
  - 取节点4的文件列表
  - 对话导出中存在、但待确认中 **没有对应文件** 的 = 未处理
  - 匹配规则：去掉方括号前缀后比较核心文件名
    - 对话导出 `[导出] 2026-03-14-1736-xxx.md` → 核心名 = `2026-03-14-1736-xxx.md`
    - 待确认 `[待确认] 2026-03-14-1736-xxx.md` → 核心名 = `2026-03-14-1736-xxx.md`
    - 没有前缀的文件（如 `cursor_04_opus_4_6_max.md`）直接用原文件名
    - 提取核心名的正则：`name.replace(/^\[.*?\]\s*/, '')`
  - 核心名相同 = 已处理，跳过
  - 返回未处理文件的列表

#### 节点6：读取原文
- 类型：`n8n-nodes-base.github`
- 操作：file → get
- 路径：`={{ $json.path }}`（来自过滤节点输出的path字段）
- base64 解码同节点2

#### 节点7：清洗组装（核心节点）
- 类型：`n8n-nodes-base.code`
- 模式：`runOnceForEachItem`

**处理逻辑（按顺序）**：

##### 7a. Base64解码
```
GitHub返回的content是base64，先解码为UTF-8文本
```

##### 7b. 提取元信息
```
从文件名和内容中提取：
- 日期：从文件名或内容中的时间戳
- 模型：搜索 "model xxx" 或 "Agent (model xxx, mode xxx)" 模式
- 用户轮次：统计 User 出现次数
- 标题：从文件名或 # 标题
```

##### 7c. 字典替换（关键！保留原文 + 标记新文）

**这是最重要的逻辑，必须严格按规则实现：**

1. 解析实体字典（从节点2获取），构建 `别名 → 标准实体` 映射表
2. 按别名长度降序排列（长的先替换，防止短别名误匹配）
3. 对正文逐个别名查找替换：
   - **原文不删除**，加删除线标记：`~~原文~~`
   - **新文紧跟其后**，用高亮标记：`==标准实体==`
   - 示例：`烂心` → `~~烂心~~==张兰心==`
   - 示例：`功法军` → `~~功法军~~==龚发军==`
   - 示例：`大睿` → `~~大睿~~==刘睿==`

4. **不替换的位置**：
   - YAML frontmatter 内不替换
   - 已经是标准实体的文本不重复替换
   - 引用块（`>`开头的行）中谨慎替换

##### 7d. 格式转换

```
原始格式（Export Transcript风格）：
  _**User (2026-03-15 14:30:25)**_
  _**Agent (model claude-4.6-opus, mode agent)**_

目标格式：
  ### 👤 用户 [14:30:25]
  ### 🤖 claude-4.6-opus [14:30:45]
```

- 删除HTML标签：`<think>...</think>`, `<tool-use>...</tool-use>`, `<details>...</details>`, `<!-- ... -->`
- 删除系统标签：`<user_query>`, `<system_reminder>`, `<image_files>` 等
- 连续空行合并为2行
- 段首段尾空白清理

##### 7e. 生成YAML头信息

```yaml
---
type: 待确认
source: Cursor/ExportTranscript
date: 2026-03-15
models: [claude-4.6-opus]
turns: 12
title: "对话标题"
original_file: "原始文件名.md"
status: pending_review
replaced_count: 15
---
```

- `replaced_count`：本次字典替换的总次数（方便用户知道要审多少）

##### 7f. 组装输出

```
输出文件名：[待确认] 原始文件名.md
输出内容：YAML头 + 格式化后的正文（含删除线+高亮标记）
```

#### 节点8：写入待确认
- 类型：`n8n-nodes-base.github`
- 操作：file → create
- 路径：`={{ '数字分身/待确认/' + $json.outputFileName }}`
- 内容：`={{ $json.outputContent }}`
- commit message：`🔍 待确认: {{ $json.outputFileName }}`

### 连接关系

```
触发 ──→ 读取实体字典 ──→ 列出对话导出 ──→ 列出待确认 ──→ 过滤未处理 ──→ 读取原文 ──→ 清洗组装 ──→ 写入待确认
```

## 工作流2：用户校对提炼

### 触发条件
用户在OB中校对完待确认文件后，手动点击执行。

### 流程图

```
手动触发
  │
  ├─→ 列出待确认/（状态=pending_review的文件）
  │
  ├─→ 列出对话存档/（已处理的文件，用于过滤）
  │
  └─→ 过滤（待确认中有、对话存档中没有的 = 用户已改未入库）
        │
        ▼
      并行读取：原始文件（对话导出/）+ 用户改版（待确认/）
        │
        ▼
      对比提炼（Code节点）
        │
        ├─→ 写入对话存档/（用户确认版，去掉标记）
        │
        └─→ 写入信号报告（对话存档/[信号] xxx.md）
```

### 核心节点规格

#### 对比提炼节点

**逻辑**：
1. 读取 `对话导出/xxx.md`（原始版）
2. 读取 `待确认/[待确认] xxx.md`（用户校对版）
3. 对比差异，提取用户动作：

| 用户动作 | 检测方法 | 含义 |
|----------|----------|------|
| 接受替换 | 原文有 `~~别名~~==标准名==`，用户版只留 `标准名` | 用户认可这个映射 |
| 拒绝替换 | 原文有 `~~别名~~==标准名==`，用户版恢复为 `别名`（删掉了标记） | 误替换，字典可能需修正 |
| 用户新增内容 | 用户版有、原始版没有的文本段落 | 用户补充了上下文/注释 |
| 用户删除内容 | 原始版有、用户版没有的段落 | 用户认为不重要或错误 |
| 用户修改YAML | YAML字段变化 | 用户纠正元信息 |

4. 生成对话存档版（去掉所有 `~~` 和 `==` 标记，保留用户最终版本的文本）：
   - YAML中 `status` 改为 `confirmed`
   - YAML中 `type` 改为 `对话存档`
   - 新增 `confirmed_date: 当前日期`
   - 文件名改为 `[存档] xxx.md`

5. 生成信号报告：

```markdown
---
type: 用户校对信号
date: 2026-03-15
source_file: "[待确认] xxx.md"
---

# 用户校对信号

## 替换结果
| 原文 | N8N替换为 | 用户动作 |
|------|-----------|----------|
| 烂心 | 张兰心 | ✅ 接受 |
| 大二 | 刘睿 | ✅ 接受 |
| xxx | yyy | ❌ 拒绝（恢复原文） |

## 用户新增内容
> （摘录用户添加的段落，如果有）

## 用户删除内容
> （摘录用户删除的段落，如果有）

## 统计
- 接受替换：X 处
- 拒绝替换：X 处
- 新增段落：X 段
- 删除段落：X 段
```

#### 写入对话存档
- 路径：`数字分身/对话存档/[存档] xxx.md`
- commit：`✅ 存档: [存档] xxx.md`

#### 写入信号报告
- 路径：`数字分身/对话存档/信号/[信号] xxx.md`
- commit：`📊 信号: [信号] xxx.md`
