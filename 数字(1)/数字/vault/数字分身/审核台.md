# 🔍 审核台

> **操作说明**：点击碎片标题打开 → 阅读内容 → 修改 frontmatter 中 `status` 字段完成审核
> - `pending` → 待审核
> - `reviewing` → 审核中（支持中途暂停，用 `🔖 审到这里` 标记位置）
> - `done` → 已完成
> - `rejected` → 丢弃 

---

## 📥 待审核（pending）

```dataview
TABLE WITHOUT ID
  file.link AS "碎片",
  source AS "来源",
  date AS "日期",
  join(entities, ", ") AS "涉及实体",
  join(tags, ", ") AS "标签"
FROM "用户输入/碎片暂存"
WHERE status = "pending"
SORT date DESC
```

## 📊 统计

```dataview
TABLE WITHOUT ID
  length(filter(rows, (r) => r.status = "pending")) AS "⏳ 待审",
  length(filter(rows, (r) => r.status = "reviewing")) AS "🔖 审核中",
  length(filter(rows, (r) => r.status = "done")) AS "✅ 完成",
  length(filter(rows, (r) => r.status = "rejected")) AS "🗑️ 丢弃",
  length(rows) AS "总计"
FROM "用户输入/碎片暂存"
FLATTEN status
GROUP BY true
```

## 📝 审核流程

### 场景 1：正常审核
1. 从「待审核」表格点击碎片标题
2. 阅读内容，有问题直接在碎片文件中批注
3. 将 `status: pending` 改为 `status: done`
4. 返回审核台，碎片自动归入「已完成」

### 场景 2：中途离开
1. 在当前阅读位置写下 `🔖 审到这里`
2. 将 `status` 改为 `reviewing`
3. 添加 `review_position` 字段记录位置
4. 下次在「审核中」表格找到继续

### 场景 3：内容有误/无价值
1. 将 `status` 改为 `rejected`
2. 可选：在文件末尾写拒绝原因

### 场景 4：需要补充信息
1. 在碎片的「待验证」区域追加问题
2. 保持 `status: reviewing`
3. 等信息补齐后再改为 `done`
