# 技术总工评估报告：Cursor对话批量导出方案

> 上级：总大脑-01
> 需求：10+窗口对话批量导出为MD，不要逐个右键

---

## 一、需求确认

| 项 | 内容 |
|---|------|
| 痛点 | 10+个Agent窗口逐个右键Export Transcript太慢、容易漏 |
| 目标 | 一键或批量导出所有/选定窗口对话为MD |
| 格式要求 | 和手动Export Transcript导出的MD格式一致（User/Cursor交替、含代码块） |
| 导出目标 | `D:\数字分身2\数字分身\对话导出\` |

---

## 二、Cursor原生能力

| 方式 | 结论 |
|------|------|
| 右键 Export Transcript | ✅ 可用，但只能逐个导出 |
| 命令面板批量导出 | ❌ 官方无此功能 |
| Cursor API | ❌ 无公开的对话导出API |
| Shared Transcripts | 🟡 可生成分享链接（50次/天限制），不是本地MD导出 |

**结论：Cursor原生不支持批量导出。必须用第三方工具。**

---

**核心功能**：

| **Export All Chats** | ✅ **批量导出多个对话** — 这就是你要的 |
| Export Chat | 单个导出 |

| List All Chats | 浏览所有对话列表 |

**导出格式**：MD / TXT(LLM优化) / JSONL 三选一

**可配置项**：

**已知限制**：

- 🔴 **格式是否和手动Export Transcript完全一致** → 未查证。从示例看格式是 `[USER]` / `[ASSISTANT]`，和Cursor原生的 `**User**` / `**Cursor**` 可能有差异

**风险**：
- 🟡 只有48安装量、0评价、v0.0.1 → 项目极早期

### 方案C：cursor-chat-export（Python CLI）

| 项 | 内容 |
|---|------|
| 来源 | GitHub: somogyijanos/cursor-chat-export |
| 查证状态 | ✅ 已查证 |
| Stars | 240（最高） |
| 平台 | Windows ✅（Python 3.12+） |
| 状态 | 🟡 已归档（2024年8月），但代码仍可用 |

**核心功能**：

| 功能 | 命令 |
|------|------|
| 发现所有对话 | `./chat.py discover` |
| 搜索特定对话 | `./chat.py discover --search-text "关键词"` |
| **批量导出全部** | `./chat.py export --output-dir "D:\数字分身2\数字分身\对话导出\"` |
| 导出指定对话 | `./chat.py export --tab-ids 2,3 --output-dir "..."` |
| 导出最近一个 | `./chat.py export --latest-tab --output-dir "..."` |

**工作原理**：直接读取 Cursor 的 `state.vscdb`（SQLite3数据库），解析对话数据，输出为MD。

**安装**：
```
git clone https://github.com/somogyijanos/cursor-chat-export.git
pip install -r requirements.txt
# 修改 config.yml 中 state.vscdb 路径为 Windows 路径
```

**风险**：
- 🔴 **格式是否和手动Export Transcript一致** → 未查证，需实测
- 🟡 项目已归档，不再维护 → 但Cursor的vscdb结构如果没大改就能用
- 🟡 需要配置 config.yml 指定正确的 vscdb 路径

## 五、评估结论

### 能解决当前痛点的方案：A 和 C

| 方案 | 能批量导出已有对话 | 安装成本 | 核心风险 |
|------|-------------------|---------|---------|
| **A: Chat Exporter** | ✅ 有Export All命令 | 低（扩展市场一键） | 🔴 48安装量/v0.0.1极早期、格式未验证 |
| **C: cursor-chat-export** | ✅ 有export全部命令 | 中（git clone + Python配置） | 🟡 已归档不维护、格式未验证 |

### 不能解决当前痛点的方案：B 和 D

| 方案 | 排除理由 |
|------|---------|
| **B: SpecStory** | 只能向前捕获新对话，不能导出已有历史。且 **已确认废弃**（CLAUDE.md+交接备忘-大脑-07明确记录"废弃SpecStory，全面删除，不再提及"） |
| **D: CursorChat Downloader** | 仅支持macOS |

### 两个方案共同的 🔴 未验证项

- 导出的MD格式是否和手动Export Transcript一致（User/Cursor交替、代码块保留）
- 能否读取到本项目当前所有窗口的对话

**以上评估供总大脑决策。技术总工不做安装/执行建议。**
