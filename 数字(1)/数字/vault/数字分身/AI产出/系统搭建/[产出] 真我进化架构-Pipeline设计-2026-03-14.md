# 真我进化架构 - Pipeline 设计

> 来源：大脑-02 与用户深度策划（2026-03-14）
> 核心思想：数字真我有两个输入源——AI做了什么 + 用户认为什么。两者交叉比对驱动进化。

---

## 三者重新定位

| 角色 | 核心问题 | 做什么 | 不做什么 |
|------|---------|--------|---------|
| **数字真我** | "我是否越来越理解你？" | 观察所有窗口产出、读用户审核信号、更新画像、识别模式、发现盲区 | 不做任务管理、不调度窗口 |
| **胜率导师** | "真我的理解对不对？" | 追踪决策三元组、用真我的理解来评估、数据说话 | 不指挥、不做事 |
| **总大脑** | "接下来做什么？" | 任务管理、窗口调度、系统运营 | 不试图深度理解用户 |

### 决策闭环

```
真我 + 用户 → 策划方向（真我更懂用户）
    ↓
形成问题 + 选项 → 告诉胜率导师
    ↓
胜率导师给客观意见（基于数据和证据）
    ↓
用户做决定 → 反馈真我
    ↓
真我把结果给胜率（追踪三元组）
```

### 模型分配

| 角色 | 模型 | 理由 |
|------|------|------|
| 数字真我 | Opus 4.6 Max | MAX给理解你的窗口 |
| 胜率导师 | Opus 4.6 Max | 独立评判，深度推理 |
| 总大脑 | Opus 4.6 | 操作调度不需要MAX |

## Pipeline 1：Cursor对话数据（先跑通这条）

```
┌─────────────────────────────────────────────────────────┐
│ 第1层：采集+清洗（N8N，已有管道）                          │
│                                                         │
│  SpecStory + 手动导出MD                                  │
│       ↓                                                 │
│  N8N 清洗管道（8节点）                                    │
│       ↓                                                 │
│  产出两份：                                               │
│  ├── 对话存档/[复盘] xxx.md（status:pending）→ OB里你审   │
│  └── 对话存档/原始/[复盘] xxx.md（永不修改，留作比对底稿）  │
└────────────────────────┬────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│ 第2层：用户审核（你在OB里操作）                            │
│                                                         │
│  读 → 留/删/标错/改 → 改status:done → commit+push        │
│  OB自带AI只是方便改材料，不是Pipeline的一部分              │
└────────────────────────┬────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│ 第3层：信号提取（N8N新管道，OpenAI比对）                   │
│                                                         │
│  N8N 定时扫描 status:done 文件                            │
│       ↓                                                 │
│  读取 done 版本 + 读取 原始/ 底稿                         │
│       ↓                                                 │
│  OpenAI 比对：用户留了什么/删了什么/标错了什么/改了什么     │
│       ↓                                                 │
│  输出：[信号] YYYY-MM-DD-xxx.md                          │
│       ↓                                                 │
│  ├── 推到 GitHub 仓库（AI产出/信号日志/）                  │
│  ├── 同步到 OB 存档                                      │
│  └── 写入向量数据库（逐条积累）                           │
└────────────────────────┬────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│ 第4层：真我进化（MAX，Cursor窗口）                        │
│                                                         │
│  真我启动 → 读信号日志 + 当前画像 → 比对 → 输出修正        │
│  画像哪里该更新？哪里被验证？哪里被推翻？                  │
└────────────────────────┬────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│ 第5层：决策闭环                                          │
│                                                         │
│  真我 + 用户 → 策划方向 → 形成问题+选项                   │
│       ↓                                                 │
│  告诉胜率导师 → 胜率给客观意见                            │
│       ↓                                                 │
│  用户做决定 → 反馈真我 → 真我把结果给胜率（追踪三元组）    │
└─────────────────────────────────────────────────────────┘
```

## 向量数据库选型

**决策：P1 用 ChromaDB**

| 理由 | 说明 |
|------|------|
| 本地运行 | 不依赖云，零成本 |
| Python原生 | `pip install chromadb` 一行启动 |
| 数据存本地 | 两地 Git 同步 |
| 扩展性 | 不够用时可换 Qdrant |

## 信号提取 OpenAI Prompt

```
你是信号提取器。你会收到两个版本的对话复盘文件：
- 【原始版】：AI清洗后的完整文稿
- 【审核版】：用户手动审核后的版本

请对比两个版本，提取以下信号：

1. 用户保留的内容（用户认可的判断/分析/建议）
2. 用户删除的内容（用户认为无价值或错误的）
3. 用户标记为AI错误的内容（AI的幻觉/误判）
4. 用户修改的内容（原文→改后，逐条列出）
5. 用户的审核模式（关注什么？忽略什么？对什么类型的AI输出最敏感？）

输出格式：
---
date: YYYY-MM-DD
source: [复盘文件名]
signals:
  kept: [列表]
  deleted: [列表]
  marked_error: [列表]
  modified: [{original: "...", changed_to: "...", interpretation: "..."}]
  pattern: [用户审核模式总结]
---
```

## Pipeline 2 预告：碎片（跑通Pipeline 1后再做）

```
钉钉转写 / 微信截图 / 会议笔记 / 随手记
    ↓
N8N 重清洗 + LLM提取（非结构化输入，比对话复盘重）
    ↓
同样的流程：进OB → 你审 → 信号提取 → 真我进化
```

碎片管道和对话管道共用第3层以后的流程（信号提取 → 向量库 → 真我进化）。
