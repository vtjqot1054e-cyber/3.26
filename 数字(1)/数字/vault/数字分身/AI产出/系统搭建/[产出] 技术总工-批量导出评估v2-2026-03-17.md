## 一、需求

| 项 | 内容 |
|---|------|
| 痛点 | 10+个Agent窗口逐个右键Export Transcript，慢、烦、容易漏 |
| 目标 | 批量导出所有/选定窗口对话为MD |
| 格式要求 | 和手动Export Transcript一致（`**User**` / `**Cursor**` 交替、含代码块） |
| 导出目标 | `D:\数字分身2\数字分身\对话导出\` |

---

## 二、Cursor原生能力

| 方式 | 结论 |
|------|------|
| 右键 Export Transcript | ✅ 可用，但只能逐个 |
| 命令面板 | ❌ 无批量导出命令 |
| API | ❌ 无公开导出API |

**结论：Cursor原生不支持批量导出。**

## 五、评估结论

**所有方案共同的 🔴 核心未验证项：导出的MD格式是否和手动Export Transcript完全一致。** 这个只能实测验证，搜索无法确认。

能批量导出已有对话的方案只有两个：**工具2（Chat Exporter扩展）** 和 **工具4（cursor-chat-export Python CLI）**。工具1（Chat Browser）的批量功能PR未合并。工具3（WayLog）只能向前捕获。

以上评估供总大脑决策。
