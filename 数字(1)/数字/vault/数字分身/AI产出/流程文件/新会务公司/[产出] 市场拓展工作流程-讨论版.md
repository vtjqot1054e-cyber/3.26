## 〇、上帝视角：这个岗位的本质

### 外部知识：BD岗位的规律

| 规律 | 来源 | 对成的意义 |
|------|------|------------|
| BD是中长期回报，不是短期 | [Kuesel Consulting](https://kueselconsulting.com/how-much-time-should-i-spend-on-business-development-theres-a-formula-for-that/) | 成的短线思维是最大障碍 |
| 时间投入要持续稳定，不能忽冷忽热 | 同上 | 需要机制强制执行 |
| 超过50小时/周生产力下降，55小时后无效 | [Stanford研究](https://www.float.com/resources/workload-management) | 工作量设计不能过载 |
| BD活动要平衡：客户维护+新开发+品牌+管道管理 | [BD School](https://www.thebdschool.com/blog/common-business-development-activities) | 不能只做擅长的 |
| 医药代表日均拜访15人是行业平均 | [医药代表日常](https://health.baidu.com/m/detail/ar_9966848635565598469) | 参考但不直接适用 |

### 成的能力边界（侧写）

| 能力 | 水平 | 工作设计的影响 |
|------|------|---------------|
| 搞关系 | ✅ 强 | 核心主线，放手让做 |
| 客户理解 | ✅ 强 | 客户分层判断靠他 |
| 洞察力 | ✅ 强 | 招投标判断靠他 |
| 行动导向 | ✅ 强 | 执行不是问题 |
| 短线思维 | ⚠️ 弱 | **需要机制强制长线动作** |
| 框架能力 | ⚠️ 弱 | 方案需要运营/老板review |
| 收尾能力 | ⚠️ 弱 | 需要催款/复盘提醒机制 |
| 战略构建 | ❌ 缺 | 方向老板定，他不参与 |
| 延迟满足 | ❌ 缺 | 不能只设长期KPI |

### 生态约束

| 约束 | 来源 | 设计影响 |
|------|------|---------|
| 口碑式传播 | 你的定位 | 核心是"做好项目带来下一个客户"，不是陌生开发 |
| 三角结构 | 行业规则 | 学会+企业+我们，成要维护三边关系 |
| 客户分6类 | 生态认知 | 不同客户不同策略 |
| 竞争对手走灰色 | 生态认知 | 我们走口碑+合规，长期优势 |
| 关系托管风险 | 核心安全问题 | 需要制度让关系沉淀到公司 |

### 核心设计原则

| 原则 | 说明 |
|------|------|
| **放开+制度控制** | 让成发挥能动性，用流程让资产沉淀到公司 |
| **扬长避短** | 强项放手做，弱项用机制补 |
| **持续稳定** | 不能忽冷忽热，每周固定动作 |
| **不过载** | 1人岗位，每周有效工作≤45小时 |

## 一、底层逻辑：口碑式传播

**我们的生意是口碑传播**。

```
客户满意（项目做得好）
    ↓
口碑传导（客户帮我们说好话）
    ↓
下一个客户
```

**市场拓展在口碑链里的位置**：
- 不是口碑的源头（源头是项目执行）
- 是口碑的**触达器**和**放大器**
- 负责让口碑传导路径清晰、持续

## 二、客户分层与策略

| 客户类型 | 真实需求 | 你应该给什么 | 接触频率 |
|----------|----------|-------------|----------|
| **学会领导**（退休厅长） | 不出事 | 安全感、合规保障、省心 | 低（项目关键节点） |
| **学会经办人**（关系户） | 少麻烦、不担责 | 省心、不让他背锅 | 中（日常维护） |
| **地方主委**（在职大专家） | 冲击全国、解决费用 | 学术价值、帮他搭台、体面 | 中（项目前后） |
| **科室主任**（学科带头人） | 完成KPI、维护圈子 | 帮他完成任务、让他体面 | 中（日常维护） |
| **秘书/科研助理** | 少干活、不出错 | 省心、不给她添麻烦 | 高（执行层对接） |
| **企业** | 合规地花钱 | 合规、有记录、能交代 | 中（项目周期） |

**你要能回答**：
1. 这个客户是哪一类？
2. 他真正要什么？
3. 谁能影响他？
4. 口碑从哪里传过来的？

## 三、主线任务（核心职责）

### 主线1：关系网络经营（每周10-12小时）

| 任务 | 频率 | 产出 | 沉淀到哪里 |
|------|------|------|-----------|
| 客户关系图谱维护 | 季度更新 | 图谱文件 | 公司系统 |
| 口碑节点识别 | 季度 | 节点清单 | 公司系统 |
| 关系强度评估 | 季度 | 评估表 | 公司系统 |
| 客情关系维护 | 每周 | 拜访/联络记录 | CRM/微信记录 |
| 转介绍引导 | 项目结束时 | 新客户线索 | 公司系统 |

**每周动作**：
- 至少联络3个关键客户（电话/微信/见面）
- 记录联络内容到系统

**设计逻辑**：成可以做关系经营，但产出必须**录入公司系统**，不能只在成脑子里。

### 主线2：招投标跟踪（每周3-4小时）

| 任务 | 频率 | 产出 |
|------|------|------|
| 信息收集 | 每周一次 | 本周招投标汇总 |
| 价值判断 | 每周 | 值不值得跟？ |
| 关系评估 | 每周 | 有没有关系？通过谁？ |
| 思路提出 | 每周 | 怎么跟？需要什么？ |
| 周会汇报 | 每周 | 市场动态简报 |

**每周动作**：
- 周一查询招投标平台
- 周五前提交市场动态简报
- 没信息也要报"本周无"

**设计逻辑**：成的短线思维容易"忘了跟"，用每周强制汇报来对抗。

### 主线3：资源转化-展商服务（项目周期，每周5-8小时）

| 任务 | 频率 | 产出 |
|------|------|------|
| 资源承接 | 项目启动时 | 企业名单 |
| 参展手册制定 | 项目启动时 | 参展手册（用模板） |
| 权益策划 | 项目启动时 | 权益方案（需review） |
| 展商服务 | 项目周期 | 解答疑问、收集需求 |
| 报价 | 项目启动时 | 报价单（大项目老板过目） |
| 催款 | 项目结束后 | 回款 |

### 主线4：团队引荐-退居二线（项目周期，每周2-3小时）

| 任务 | 频率 | 产出 |
|------|------|------|
| 引荐客户经理组长 | 项目启动时 | 交接完成 |
| 引荐项目经理组长 | 项目启动时 | 交接完成 |
| 引荐运营 | 项目启动时 | 交接完成 |
| 交接客户背景 | 引荐时 | 交接记录（用模板） |
| 退居二线 | 引荐后 | 只处理客情、投诉、危机 |

**设计逻辑**：引荐后，客户经理组长直接对接客户，关系在客户经理组长那里积累。成的价值是**开门**，不是**守门**。

## 五、每周时间分配（工作量评估）

### 正常周（无大型项目）

| 任务类型 | 时间/周 | 说明 |
|----------|---------|------|
| 关系网络经营 | 10-12小时 | 客户联络、关系维护 |
| 招投标跟踪 | 3-4小时 | 查询、分析、汇报 |
| 品牌推广配合 | 1-2小时 | 素材、机会识别 |
| 竞争对手监控 | 0.5小时 | 分摊到每周 |
| 内部协调/会议 | 3-4小时 | 周会、沟通 |
| **合计** | **18-23小时** | 约2.5-3天 |

### 项目周（有大型项目进行中）

| 任务类型 | 时间/周 | 说明 |
|----------|---------|------|
| 关系网络经营 | 8-10小时 | 压缩但不能停 |
| 招投标跟踪 | 2-3小时 | 压缩但不能停 |
| 资源转化-展商服务 | 8-12小时 | 项目相关 |
| 团队引荐-退居二线 | 3-5小时 | 交接、客情 |
| 品牌推广配合 | 1小时 | 压缩 |
| 内部协调/会议 | 4-5小时 | 项目会议增加 |
| **合计** | **26-36小时** | 约4-5天 |

### 高峰周（多个项目叠加）

| 任务类型 | 时间/周 | 说明 |
|----------|---------|------|
| 项目相关工作 | 25-30小时 | 多项目叠加 |
| 关系+招投标 | 8-10小时 | 最低限度维持 |
| 内部协调 | 5-6小时 | 协调量增加 |
| **合计** | **38-46小时** | 接近饱和 |

### 工作量评估结论

| 评估维度 | 结论 |
|----------|------|
| **可行性** | ✅ 1人可承担，但高峰期会满负荷 |
| **瓶颈** | 多项目叠加时，关系维护容易被压缩 |
| **风险** | 成的短线思维+高峰期压缩 = 长线任务容易断 |
| **对策** | 用机制强制长线任务不能停（每周简报、季度图谱） |

## 七、关系资产沉淀机制（制度控制）

**核心问题**：成可以经营关系，但关系资产必须沉淀到公司，不是沉淀到成个人。

| 成做的事 | 必须录入的地方 | 谁检查 |
|----------|---------------|--------|
| 客户联络 | 客户联络记录表/CRM | 运营 |
| 客户背景了解 | 交接记录模板 | 客户经理组长 |
| 口碑节点识别 | 客户关系图谱 | 老板 |
| 招投标判断 | 市场动态简报 | 老板 |

**机制**：
- 没有录入 = 没有做
- 季度和老板过图谱，检查关系资产是否在系统里

## 八、风险控制机制（8项）

### 1. 客户关系图谱机制（对抗关系托管风险）
- 季度更新，录入公司系统
- 和老板过一遍

### 2. 招投标跟踪机制（对抗短线思维）
- 每周强制汇报
- 没信息也要报"本周无"

### 3. 信息回流机制（对抗信息孤岛）
- 客户经理组长每周简报给市场拓展
- 异常立即通知

### 4. 长线跟踪提醒机制（对抗短线思维）
- 冷淡客户每2周提醒
- 运营负责提醒

### 5. 过度承诺约束机制（对抗过度承诺）
- 超范围/赔偿/大幅让步需老板授权

### 6. 报价验证机制（对抗报价失误）
- 大项目老板过目

### 7. 催款跟踪机制（对抗收尾弱）
- 项目结束进催款表
- 到期提醒

### 8. 丢单复盘机制（对抗信息缺失）
- 每次丢单分析原因
- 季度汇总

## 九、介入场景（4种）

### 场景1：客户高层首次见面
- 你：开场、破冰
- 客户经理组长：了解需求
- 后续：客户经理跟进

### 场景2：重要提案/汇报
- 你：暖场、收尾
- 客户经理组长/项目经理组长：汇报方案
- 注意：会上承诺超方案 → 会后对齐

### 场景3：客户投诉/危机
- 客户经理：第一时间响应
- 客户经理组长：判断是否需要你
- 你：灭火沟通
- 注意：赔偿需老板授权

### 场景4：日常对接
- 客户经理组长/项目经理组长/运营直接对客户
- 你不主动介入

## 十二、总结

### 这个岗位的本质

```
成 = 口碑传导的触达器和放大器
      ↓
通过关系网络经营，让口碑传导路径清晰、持续
      ↓
关系资产通过制度沉淀到公司，不是沉淀到成个人
```

### 设计逻辑

| 设计 | 理由 |
|------|------|
| 让成做关系经营 | 发挥能动性，这是他的强项 |
| 用机制强制长线动作 | 对抗短线思维 |
| 产出必须录系统 | 关系资产沉淀到公司 |
| 引荐后退居二线 | 关系在客户经理组长那里积累 |
| 工作量控制在45小时/周以内 | 1人可持续 |

### 关键成功因素

1. **成愿意录系统**——需要老板推动
2. **每周简报执行**——需要形成习惯
3. **季度图谱检查**——需要老板亲自参与
4. **引荐后真的放手**——需要老板观察
