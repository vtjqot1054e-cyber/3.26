> 创建日期：2026-02-01

## 〇、上帝视角：这个岗位的本质

### 外部知识：会务运营岗位的规律

| 规律 | 来源 | 对运营策划的意义 |
|------|------|------------|
| 会务运营需要事前-事中-事后全流程管理 | [汉坤律师事务所](https://www.hankunlaw.com/portal/article/index/cid/8/id/14124.html) | 擅长流程管理，匹配 |
| 合规文件需要妥善保管相关资料 | 同上 | 需要强制归档机制 |
| 数字化管理可提升协同效率48% | [简道云活动策划](https://www.jiandaoyun.com/news/article/68ac1e1e229b892d522c8ae9) | 应主导系统建设 |
| 进度追踪需明确责任人和时间节点 | 同上 | 强项是执行章法 |
| 招投标电子化监管趋势加强 | [锦天城律所](https://www.allbrightlaw.com/CN/10475/84a977341e07f79b.aspx) | 合规要求越来越严格 |
| 复盘机制是运营核心能力 | [简道云](https://www.jiandaoyun.com/nblog/399370/) | 不复盘是最大问题 |

### 岗位能力要求

| 能力 | 要求 | 工作设计的影响 |
|------|------|---------------|
| 逻辑能力 | ✅ 必须强 | 投标应答、文件编制核心能力 |
| 执行章法 | ✅ 必须强 | 流程设计、进度追踪核心能力 |
| 技术问题解决 | ✅ 必须强 | 系统建设、工具使用核心能力 |
| 思维深度 | ✅ 必须强 | 因果链清晰 |
| 市场感觉 | ❌ 不需要 | **商业判断归市场拓展** |
| 客户性格处理 | ❌ 不需要 | **客户关系归市场拓展** |
| 知识分享 | ⚠️ 需强制 | 需要强制输出机制，防止知识垄断 |
| 复盘 | ⚠️ 需强制 | 需要强制复盘机制 |

### 生态约束

| 约束 | 来源 | 设计影响 |
|------|------|---------|
| 合规要求越来越严 | 医药反腐 | 合规工作是核心价值 |
| 政府补贴需要客户配合 | 行业规则 | 需要市场拓展引荐运营策划给客户 |
| 招投标需要合规操作 | 法律要求 | 运营策划负责文件编制，市场拓展负责商业判断 |
| 老板需要审批预算 | 公司规则 | 运营策划整理初稿，老板签字 |

### 核心设计原则

| 原则 | 说明 |
|------|------|
| **市场拓展+运营策划一体** | 前台开拓+后台支撑，是一个整体 |
| **在背后做事** | 不直接面对客户关系问题，通过引荐建立连接后对接具体事务 |
| **任务驱动** | 减少选择性做事的空间，用清单+节点管控 |
| **强制输出** | 对抗知识垄断，所有产出必须文档化 |
| **老板支撑** | 策划框架老板定，运营策划填充细节 |
| **不管人** | 运营策划不负责管理团队，只负责流程 |

## 一、岗位定位

**运营策划是市场拓展的后台支撑，公司的流程守护者和合规保障者。**

```
运营策划的价值
    │
    ├── 合规保障：补贴申报、安保消防、合规备案
    ├── 进度追踪：追踪项目进度、催款进度、任务进度
    ├── 预算管控：预算初稿整理、成本控制
    ├── 投标支撑：投标文件编制、应答策略
    ├── 策划支撑：方案框架填充、招商手册制定
    └── 系统建设：流程沉淀、模板建设、工具推广
```

**核心定位**：和市场拓展是一个整体，在背后做事，用逻辑和流程支撑前线。

## 二、与其他角色的边界

| 职责 | 运营策划 | 市场拓展 | 老板 |
|------|-----------|---------------|------|
| 投不投标 | ❌ 不判断 | ✅ 商业判断 | 大项目审核 |
| 报多少价 | ❌ 不判断 | ✅ 定价 | 大项目审核 |
| 怎么投标 | ✅ 文件编制、应答策略 | 提供客户信息 | — |
| 补贴申报 | ✅ 主责 | 配合（关系资源） | — |
| 安保消防 | ✅ 主责 | — | — |
| 预算整理 | ✅ 初稿 | 提供价格信息 | 签字 |
| 进度追踪 | ✅ 主责 | 接收提醒 | — |
| 策划框架 | ✅ 填充细节 | — | 大型项目定方向 |
| 招商手册 | ✅ 制定 | 提供权益信息 | — |

**关键边界**：
- 运营策划不做商业判断（市场拓展的职责）
- 运营策划不直接处理客户关系问题
- 运营策划不管人，只负责流程

## 三、主线任务

### 主线1：合规报备（政府补贴+安保消防+合规备案）

**目标**：确保项目合规运行，拿到该拿的补贴。

#### 3.1.1 政府补贴申报

**流程**：
```
项目确定 → 确认是否有补贴政策
    ↓
有 → 整理申报材料
    ↓
市场拓展引荐运营策划给客户
    ↓
运营策划对接客户收集材料（公函、资质、证明）
    ↓
运营策划整理申报文件
    ↓
提交申报
    ↓
跟踪进度 → 拨付到账
    ↓
归档
```

**需要客户提供的材料**：
| 材料类型 | 说明 |
|----------|------|
| 主办方公函 | 证明会议由学会/机构主办 |
| 参会证明 | 签到表、现场照片等 |
| 财务材料 | 发票、合同等 |
| 资质证明 | 营业执照等 |

**产出**：补贴申报文件、申报记录

**频率**：按项目

**配合**：
- 市场拓展：引荐张给客户、协调关系资源
- 客户：提供材料

---

#### 3.1.2 安保消防报备

**流程**：
```
项目确定场地 → 确认报备要求
    ↓
整理报备材料
    ↓
项目经理组长提供场地图、现场信息
    ↓
提交报备
    ↓
跟踪审批
    ↓
获批后归档
```

**需要的材料**：
| 材料类型 | 谁提供 |
|----------|--------|
| 场地平面图 | 项目经理组长 |
| 活动方案 | 客户经理组长 |
| 人数预估 | 客户经理组长 |
| 应急预案 | 运营策划整理 |

**产出**：报备文件、批复文件

**频率**：按项目

**配合**：
- 项目经理组长：场地图、现场信息
- 客户经理组长：活动方案、人数

---

#### 3.1.3 合规备案

**流程**：
```
项目启动 → 确认备案要求
    ↓
收集备案材料
    ↓
整理备案文件
    ↓
提交备案
    ↓
跟踪备案状态
    ↓
备案完成 → 归档
```

**产出**：备案文件、备案证明

---

### 主线2：预算管控

**目标**：控制成本，确保利润。

#### 3.2.1 预算初稿整理

**流程**：
```
项目确定 → 收集各组预算数据
    ↓
市场拓展：招商收入预估、总体价格
客户经理组长：接待统筹、宣传预算
项目经理组长：搭建设备、视频预算
设计组长：设计物料预算
    ↓
运营策划整理预算初稿
    ↓
老板审批签字
    ↓
执行
    ↓
项目结束 → 决算对比 → 差异分析
```

**产出**：预算初稿、决算报告

**频率**：每个项目

---

#### 3.2.2 成本跟踪

**流程**：
```
项目执行中 → 各组提报实际支出
    ↓
运营策划汇总 → 与预算对比
    ↓
超预算 → 预警 → 找原因 → 上报老板
    ↓
项目结束 → 出决算 → 和预算对比 → 归档
```

**产出**：成本跟踪表、决算报告

---

### 主线3：进度追踪

**目标**：确保项目按时推进，不遗漏关键节点。

#### 3.3.1 项目进度追踪

**流程**：
```
项目启动 → 运营策划建立项目进度表
    ↓
分解关键节点、责任人、截止时间
    ↓
每周更新进度状态
    ↓
到期未完成 → 提醒责任人
    ↓
风险预警 → 汇报老板
    ↓
项目结束 → 归档
```

**产出**：项目进度表、进度周报

**频率**：每周更新

---

#### 3.3.2 催款追踪

**流程**：
```
项目结束 → 市场拓展录入催款信息
    ↓
运营策划维护催款跟踪表
    ↓
到期未回款 → 提醒市场拓展
    ↓
超期30天 → 升级老板
```

**产出**：催款跟踪表

**频率**：每周检查

**配合**：
- 市场拓展：跟进催款、录入信息
- 运营策划：提醒、跟踪、升级

---

#### 3.3.3 长线客户提醒

**流程**：
```
市场拓展更新客户关系图谱
    ↓
运营策划根据图谱设置跟进节点
    ↓
客户冷淡/长期未联络 → 提醒市场拓展
```

**产出**：客户跟进提醒

**频率**：每周检查

---

### 主线4：投标支撑

**目标**：用逻辑和专业支撑投标工作。

#### 3.4.1 投标文件编制

**流程**：
```
市场拓展确定要投的项目
    ↓
市场拓展提供：客户信息、报价、投标策略
    ↓
运营策划编制投标文件：
    ├── 商务标
    ├── 技术标
    ├── 资质证明
    └── 应答文件
    ↓
老板审核（大项目）
    ↓
提交投标
    ↓
开标 → 中标/未中标 → 归档
```

**产出**：投标文件

**频率**：按项目

---

#### 3.4.2 投标应答策略

**运营策划的职责**：
- 分析招标文件要求
- 设计应答框架
- 确保合规

**不是运营策划的职责**：
- 判断值不值得投（市场拓展判断）
- 定价（市场拓展判断）

---

#### 3.4.3 投标档案管理

**流程**：
```
每次投标结束 → 归档
    ↓
归档内容：招标文件、我方投标文件、开标记录、中标/落标通知
    ↓
落标 → 分析原因（配合市场拓展）
```

**产出**：投标档案

---

### 主线5：策划支撑

**目标**：支撑方案策划工作。

#### 3.5.1 策划框架填充

**分工**：
| 项目类型 | 策划框架谁定 | 运营策划做什么 |
|----------|-------------|---------|
| 大型项目 | 老板 | 填充细节、整理文档 |
| 中小项目 | 运营策划 | 出初稿，老板审核 |

**流程**：
```
大型项目：
老板定策划框架/方向
    ↓
运营策划填充细节
    ↓
整理成策划文档
    ↓
客户经理组长/项目经理组长拿去对接

中小项目：
运营策划出策划初稿
    ↓
老板审核
    ↓
客户经理组长对接
```

**产出**：策划文档

## 四、支线任务

### 支线1：系统建设

**目标**：用系统沉淀知识，对抗知识垄断。

#### 4.1.1 模板库建设

**做什么**：建立并维护常用模板库。

**模板类型**：
| 模板 | 用途 |
|------|------|

| 补贴申报模板 | 补贴申报 |

| 策划框架模板 | 策划 |
| 会议纪要模板 | 会议 |

**产出**：模板库

**频率**：持续更新

---

#### 4.1.2 流程沉淀

**做什么**：将经验转化为可复用的SOP。

**产出**：流程文档库

**频率**：每季度更新

---

#### 4.1.3 工具推广

**做什么**：推广数字化工具，提升效率。

**工具类型**：
| 工具 | 用途 |
|------|------|
| 项目管理工具 | 进度追踪 |

| 文档协作工具 | 团队协作 |

### 支线2：数据分析

**目标**：用数据支撑决策。

#### 4.2.1 丢单分析配合

**做什么**：配合市场拓展分析丢单原因。

**产出**：丢单分析汇总

**频率**：每季度

### 支线3：复盘推动

|------|------|

| 内部协调/会议 | 5-6小时 | 会议增加 |
| **合计** | **27-39小时** | 约4-5天 |

### 高峰周（投标密集+多项目）

| 任务类型 | 时间/周 | 说明 |
|----------|---------|------|
| 投标支撑 | 15-20小时 | 多个投标 |
| 项目相关 | 15-20小时 | 多项目 |
| 内部协调 | 5-6小时 | |
| **合计** | **35-46小时** | 接近饱和 |

### 工作量评估结论

| 评估维度 | 结论 |
|----------|------|
| **可行性** | ✅ 1人可承担，高峰期满负荷 |
| **瓶颈** | 投标密集期，其他工作会被压缩 |
| **风险** | 选择性做事+高峰期 = 非核心任务被放弃 |
| **对策** | 用任务清单强制追踪，减少选择空间 |

## 七、风险控制机制（6项）

### 1. 任务清单机制（对抗选择性做事）

**问题**：可能选事情做，不重要的就不做。

**机制**：
```
每周一 → 运营策划提交本周任务清单
周五 → 检查完成情况
未完成 → 说明原因 → 调整优先级
```

**执行**：老板周一审核任务清单，周五检查完成情况。

---

### 2. 强制文档化机制（对抗知识垄断）

**问题**：可能把知识锁在脑子里，不分享。

**机制**：
```
每个核心流程 → 必须有文档
每次投标 → 归档完整文件
每次复盘 → 有复盘记录
```

**检查**：每季度检查文档库完整性。

---

### 3. 强制复盘机制（对抗不复盘）

**问题**：丢了就丢了，不研究。

**机制**：
```
每个重要项目结束 → 必须复盘
每次投标落标 → 必须分析原因
复盘记录 → 归档
```

**执行**：老板强制要求。

---

### 4. 进度可视化机制（对抗信息黑洞）

**问题**：进度在张脑子里，别人不知道。

**机制**：
```
项目进度表 → 全员可见
周报 → 每周提交
异常 → 立即通知
```

---

### 5. 引荐后对接机制（对抗客户直接对接困难）

**问题**：运营策划不直接处理客户关系问题，但合规工作需要客户配合。

**机制**：
```
市场拓展引荐运营策划给客户（建立连接）
    ↓
运营策划对接具体事务（材料收集等）
    ↓
客户有情绪问题 → 市场拓展介入
```

---

### 6. 商业判断禁止机制（对抗越界）

**问题**：运营策划可能越界做商业判断，但这是市场拓展的职责。

**机制**：
```
投不投 → 市场拓展判断
报价 → 市场拓展定
运营策划只负责 → 怎么投、文件编制
```

**边界**：运营策划不能独立决定投不投、报多少。

## 十一、总结

### 这个岗位的本质

```
运营策划 = 市场拓展的后台支撑 + 公司的流程守护者和合规保障者
        ↓
    和市场拓展是一个整体
        ↓
    在背后做事，用逻辑和流程支撑前线
        ↓
    发挥强项（逻辑、执行章法、策划支撑）
        ↓
    避开弱项（客户关系、商业判断、管人）
        ↓
    用机制对抗问题（知识垄断、选择性做事、不复盘）
```

### 设计逻辑

| 设计 | 理由 |
|------|------|
| 和市场拓展一体 | 前台开拓+后台支撑，互相配合 |
| 在背后做事 | 不直接处理客户关系问题 |
| 不做商业判断 | 商业判断是市场拓展的职责 |
| 不管人 | 只负责流程，不负责管理团队 |
| 任务驱动 | 对抗选择性做事 |
| 强制文档化 | 对抗知识垄断 |
| 强制复盘 | 对抗不复盘 |
| 老板支撑 | 策划框架需要老板定 |

### 关键成功因素

1. **市场拓展+运营策划紧密配合** —— 是一个整体
2. **老板定期检查任务清单** —— 否则会选事情做
3. **强制文档化执行** —— 否则知识锁在脑子里
4. **强制复盘执行** —— 否则同样的错重复犯
5. **商业判断边界明确** —— 运营策划不越界
