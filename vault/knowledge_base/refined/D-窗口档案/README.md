# D-窗口档案

> 数字真我管理，胜率之我审计。
> 每个窗口角色一个档案文件，记录该角色的行为特性、历史表现、常见偏差。
> 核心逻辑：每个窗口的"人"= 岗位职能 + 用户在该场景下的行为模式。屁股决定脑袋。

## 档案结构（每个角色一个文件）

```
D-窗口档案/
├── README.md（本文件）
├── 复盘大脑.md
├── 总顾问大脑.md
├── 技术总工.md
├── 工程.md
└── ...按需新增
```

## 每个档案包含

1. **角色定位**：岗位职能、模型、层级
2. **行为特性**：用户在该角色场景下的行为模式（和其他窗口的差异）
3. **历史表现**：任务完成率、常见问题、纪律记录
4. **常见偏差**：该角色容易犯的错（AI侧+用户侧）
5. **因人施策**：针对该角色特性的管理建议
6. **任务追踪**：当前未完成任务、历史任务记录

## 数据来源

- 审计意见与纪律记录.md（按篇记录 → 按角色归档）
- 各窗口交接备忘中的表现记录
- 复盘大脑终审报告中的AI行为分析
