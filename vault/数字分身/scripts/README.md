# 用户发言提取脚本 - 使用说明

## 文件位置

- **Python 脚本**: `vault/数字分身/scripts/extract_user_messages.py`
- **快捷批处理**: `vault/数字分身/scripts/提取用户发言.bat`

## 快速使用 (推荐)

**双击运行** `提取用户发言.bat`:
1. 会自动查找最新的 SpecStory 文件
2. 提示你输入版本名称(如: V18)
3. 自动提取并生成待定性文件

## 命令行使用

### 方式1: 自动模式(推荐)
```powershell
python extract_user_messages.py auto "V18"
```
自动查找 `e:\数字\.specstory\history` 下最新的文件

### 方式2: 指定文件
```powershell
python extract_user_messages.py "e:\数字\.specstory\history\2026-04-01_xxx.md" "V18"
```

## 输出文件

生成到: `vault/数字分身/用户输入/待定性/[待定性] V18-2026-04-01.md`

**格式示例**:
```markdown
# V18 用户发言待定性

1、"入库,根据这个文档复盘系统"
2、"不会影响我们的进程吧?"
3、"给工程指令吧"

---
**定性规则**:
- 什么都不写 = 中性
- 写「好」= 好
- 写「坏」= 坏
- 可以在序号后加备注

**示例**:
5、坏,AI没执行关窗规范
6、坏
```

## 定性流程

1. 运行脚本生成待定性文件
2. 打开文件,在序号后添加定性标记
3. 关窗时 AI 读取并消费
4. 处理完的文件会移动到 `已定性/` 目录

## 提取规则

脚本会提取以下标记后的内容:
- `**User**` 后的内容
- `### 👤 用户` 后的内容

自动去重,相同内容只保留一条。

## 故障排查

- **找不到文件**: 检查 `e:\数字\.specstory\history` 路径是否存在
- **编码错误**: 脚本使用 UTF-8 编码,确保 SpecStory 文件也是 UTF-8
- **Python 版本**: 需要 Python 3.6+
