## 二、AutoResearch 工作模式

### 基本信息

- **发起人**：Andrej Karpathy
- **发布时间**：2026年3月
- **定位**：自主改进闭环模式（Autonomous Improvement Loop）

### 核心循环

```
提出变更（LLM Agent 提议）
      ↓
运行实验（自动执行）
      ↓
评估结果（fitness 函数打分）
      ↓
保留/回滚（好的保留，差的撤销）
      ↓
迭代（带着更新状态进入下一轮）
```

### 关键特点

- **效率**：每小时约 12 次实验，一夜 100+ 次，约 20x 人工提速
- **人机分工**：人类只设定目标和边界，AI 负责迭代执行
- **核心洞察**：不是一次性任务，而是观察结果并适应性调整
- **Checkpoint**：每轮保存状态，防止崩溃丢失进展

### 适用场景

- ML 超参数优化
- GPU kernel 优化
- 交易策略回测
- 代码性能优化
- 任何**可量化评估**的系统

## 三、AutoResearch vs 数字真我：结构对比

| 维度 | AutoResearch | 数字真我 | 对齐程度 |
|------|-------------|---------|---------|
| **核心循环** | 提议 → 实验 → 评估 → 保留/撤销 | 判断 → 碰撞 → 共鸣/纠偏 → 修正 | ✅ 结构高度一致 |
| **校准信号** | Fitness 函数（量化指标） | 用户的共鸣/纠偏（定性反馈） | ⚠️ 信号类型不同 |
| **迭代速度** | 全自动，每小时 12 次 | 依赖人机对话节奏 | ⚠️ 我们受限于碰撞需人参与 |
| **状态保持** | Checkpoint 机制 | 知识库 + 交接备忘 | ✅ 已有类似设计 |
| **退化检测** | Fitness 下降就撤销 | AI 无法自检，用户是传感器 | ⚠️ 我们缺少量化退化指标 |
| **跨轮延续** | 读 history + checkpoint 恢复 | 读 SK + 知识库 + 上轮共鸣 | ✅ 理念一致 |
| **约束机制** | 预设边界（单脚本、参数空间） | 防平庸规则、约束性结论 | ✅ 已对齐 |

## 四、可借鉴点

### 1. Fitness 函数思路 → 螺旋点硬目标

**AutoResearch 做法**：用量化的 fitness 函数判断每轮是进步还是退步。

**借鉴**：每个窗口关闭时，记录可量化指标：
- 知识库新增条数
- 共识/共鸣数量
- 画像更新条数
- 系统问题解决数

用于判断这轮是「螺旋上升」还是「原地转」。

### 2. Checkpoint 机制 → 强化关窗流程

**AutoResearch 做法**：每轮保存状态，崩溃可恢复。

**借鉴**：把关窗流程从「规范要求」升级为「系统强制」：
- MCP 检索关窗模板
- 自动落盘到交接备忘
- 不依赖模型「记住关窗规范」

### 3. 保留/撤销机制 → 认知版本管理

**AutoResearch 做法**：发现退化（fitness 下降）就回滚到上一个好状态。

**借鉴**：给关键认知加「试用期」标签：
- 新沉淀的认知先打「待验证」标签
- 经过 3 次以上碰撞确认后转正
- 如果后续碰撞否定，可标记废弃而非删除

### 4. 退化预警机制

**AutoResearch 做法**：fitness 下降触发自动回滚。

**借鉴**：设计「共鸣频率下降」预警：
- 连续 N 轮没有共鸣 → 警告可能在原地转
- 纠偏频率突增 → 警告可能偏离轨道
- 用户主动喊停 → 立即进入复盘

## 五、不匹配的地方（边界）

| AutoResearch 特性 | 数字真我现状 | 差异原因 |
|------------------|-------------|---------|
| 全自动迭代 | 需要用户参与碰撞 | **碰撞是认知的源头**，不能自动化 |
| 量化 fitness | 主要靠定性共鸣 | 认知校准难以纯量化 |
| 单一目标优化 | 多维度认知积累 | 我们不是在优化一个指标 |
| 失败可撤销 | 沉淀后难以回滚 | 认知入库后混入其他引用 |

**核心差异**：AutoResearch 的价值在于「自动化跑实验」，数字真我的价值在于「碰撞产生理解」。循环结构可借鉴，但校准信号不能照搬。

## 六、落地建议

### 短期可执行

1. **螺旋点量化**
   - 在关窗流程中增加「本轮量化指标」字段
   - 字段包括：新增知识条数、共鸣次数、纠偏次数、系统问题解决数

2. **退化预警**
   - 参考 fitness 下降触发回滚的思路
   - 设计「共鸣频率下降」→「停下来复盘」机制

3. **认知试用期**
   - 新沉淀的认知先打「待验证」标签
   - 3 次以上碰撞确认后转正

### 中期探索

4. **关窗强制化**
   - 把关窗模板检索写入 SK 或 CLAUDE.md
   - 探索 Cursor CLI / n8n 实现半自动落盘

5. **认知版本管理**
   - 给知识库条目加版本号或时间戳
   - 支持「废弃」标记而非硬删除

## 八、结论

AutoResearch 与数字真我的**螺旋式校准**在结构上高度一致：都是「提议 → 执行 → 评估 → 保留或修正 → 迭代」的闭环。

**可借鉴**：
- Checkpoint / 状态保持 → 强化关窗流程
- Fitness 量化 → 螺旋点硬目标
- 退化回滚 → 认知试用期 + 退化预警

**不可照搬**：
- 全自动迭代（碰撞需要人参与）
- 纯量化评估（认知校准是定性的）

核心洞察：**循环结构借鉴，校准信号保留人机碰撞**。
