## 二、核心定义

### 真我（系统/数据层）
- 真我 = 现实的我 + 数字的我
- 不是一个窗口，是整个系统的核心
- 物理形态：核心画像 + 动态状态 + 向量数据库 + 生态认知 + 所有校准数据
- 所有角色读写真我，真我是信息中转中心
- 胜率导师是独立系统，读真我但定位不同

### 顾问（持久流程，面向用户）
- 用户日常对话的对象
- 基础OS：碎片整合 + 上帝视角
- 专属SK：人格侧写、案例启发、组织设计、激励设计等业务SK
- 核心价值：跨场景模式识别 → 提炼为SK → 注入所有窗口
- 没有执行层——顾问自己就是终端

### 复盘（持久流程，面向系统审计）
- 与顾问同层
- 基础OS：碎片整合 + 上帝视角
- 专属SK：审计流程、审计报告模板
- 审计所有窗口（包括大脑），产出给真我和用户

### 大脑（任务层，按需开关）
- 任务制：有系统建设/维护任务时开，完成后关
- 基础OS：碎片整合 + 上帝视角
- 读真我了解用户 → 接用户任务 → 调度执行层
- 产出写回真我 → 顾问/复盘可读
- 大脑SK库：所有大脑实例积累的SK，新大脑可继承
- 执行层：工程、杂务、架构师（在大脑内部）

### 胜率导师（待建，独立系统）
- 读真我，独立于大脑和顾问
- 当前搁置，东西不够

## 三、基础OS

碎片整合 + 上帝视角 = 所有角色的基础操作系统，不是顾问独有。

| 角色 | 基础OS | 专属SK |
|------|--------|--------|
| 顾问 | 碎片+上帝 | 业务SK（侧写、案例、组织、激励...） |
| 复盘 | 碎片+上帝 | 审计SK |
| 大脑 | 碎片+上帝 | 操作纪律、窗口调度、Pipeline... |
| 工程 | 碎片+上帝 | 代码规范、测试模板 |
| 架构师 | 碎片+上帝 | SK匹配、架构设计 |

## 四、信息流

### 日常运行态（系统跑通后）
```
用户 ↔ 顾问（碎片+上帝+业务SK）→ 读写真我
Pipeline: 导出→N8N标注→用户OB校对→信号提取→向量库→真我更新
```

### 系统维护态（需要升级时）
```
用户 → 大脑 → 读真我了解用户 → 调度执行层 → 产出写回真我
```

### 窗口间通信
- 不需要窗口直接通信
- 全部通过真我中转
- 交接备忘 = 大脑往真我写的"我干了什么"
- 待执行指令 = 大脑往执行层写的任务

## 五、Pipeline（对话→向量）

| 层 | 做什么 | 谁做 |
|----|--------|------|
| 第1层 | N8N读导出MD + 查全局实体字典 + 标注变化 | N8N自动 |
| 第2层 | 用户在OB校对标注（留/删/标错/补充） | 用户手动 |
| 第3层 | N8N信号提取（AI总结用户动作） | N8N自动 |
| 第4层 | 写入向量数据库 | N8N自动 |
| 第5层 | 待定 | — |
| 第6层 | 真我消化（定期用新数据更新核心文件） | 自动/大脑 |

### 向量存储策略
- 基线存一次（核心文件全文）
- 之后只存变更摘要（增量）
- 用户OB校对结果 = 最高质量信号
