## 4. 推荐方案：三目录联合排除

### WF1过滤逻辑改造

当前（只查1个目录）：
```
对话导出coreName ∈ 待确认/ → 跳过
```

改为（查3个目录）：
```
对话导出coreName ∈ 待确认/ → 跳过（审阅中）
对话导出coreName ∈ 对话存档/（通过frontmatter original_file匹配）→ 跳过（已归档）
对话导出coreName ∈ 对话删除/ → 跳过（已取消）
都不在 → 生成新的待确认
```

### 改动清单

**改动1：WF1 `列出待确认` 节点**
- 扩展为同时读 `待确认/`、`对话删除/` 两个目录的文件名

**改动2：WF1 新增 `列出对话存档` 节点**
- 读 `对话存档/` 所有MD文件的frontmatter `original_file` 字段
- 提取coreName加入排除集合

**改动3：WF1 `过滤未处理` 节点**
- 合并三个目录的排除集合做过滤

**改动4：新增WF4（cancelled → 对话删除）**
- 扫 `待确认/` → 读frontmatter → status=cancelled → 移动到 `对话删除/` → 删除待确认源文件

### 对话存档匹配逻辑

对话存档文件名（`[对话] 2026-03-17-0056-xxx.md`）和对话导出coreName无法直接匹配，需通过frontmatter反查：

```
对话存档 frontmatter → original_file: "[导出] 2026-03-18-2206-xxx.md"
  → 去前缀提取coreName: "2026-03-18-2206-xxx.md"
  → 匹配对话导出coreName → 排除
```
