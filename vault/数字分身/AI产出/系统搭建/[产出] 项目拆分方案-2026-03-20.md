# 数字分身×数字真我 物理拆分方案

## 决策背景

- 两个系统混在同一个项目文件夹，共用CLAUDE.md和.cursor/skills/
- 调试过程中已发生互相修改配置的情况
- 11篇归档完成，管线冻结条件（10篇）已满足
- 需要物理隔离以支持各自独立演化

## 已确认的决策

| 决策项 | 结论 | 谁确认 |
|--------|------|--------|
| 一个vault两个顶级目录 | ✅ | 数字真我+大脑+用户 |
| 一个Git仓库不拆 | ✅ | 数字真我+大脑+用户 |
| vault根新建vault/子目录 | ✅ | 大脑提出，数字真我同意 |
| 核心画像放数字真我 | ✅ | 数字真我提出，大脑同意 |
| Cursor项目独立 | ✅ | 用户决定 |
| 交接备忘各自独立 | ✅ | 用户决定 |
| N8N挂载关系暂不变 | ✅ | 用户 |
| 家里先做 | ✅ | 用户 |

## 风险及应对

| 风险 | 应对 |
|------|------|
| OB缓存/索引要重建 | 打开新vault后OB会自动重建索引 |
| wikilink里用了相对路径 | OB的wikilink是按文件名匹配的，不受目录层级影响，除非有同名文件 |
| 同名文件冲突 | 检查：交接备忘.md 两边各一份，OB wikilink需要用完整路径 `[[数字真我/交接备忘]]` |
| .specstory历史数据 | 留在仓库根不动，新的Cursor项目会各自生成新的.specstory |
| 定时git push脚本路径 | 脚本在仓库根运行git命令，不受vault层影响 |
| pip install chromadb | 家里环境暂缓，拆分完后再装 |
