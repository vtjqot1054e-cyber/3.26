# 市场拓展工作流程

> 创建日期：2026-02-01
> 适用岗位：市场拓展

---

## 一、岗位定位

**你是公司的资源入口和客户桥梁。**

```
你的价值
    │
    ├── 资源承接：把科室、学会、企业的资源接进来
    ├── 客户桥梁：把客户和团队连接起来
    ├── 口碑放大：让做得好的项目带来下一个客户
    └── 客情兜底：关键时刻出面解决问题
```

**我们的生意是口碑传播**——项目做得好，客户帮我们说好话，下一个客户就来了。你的工作就是让这个链条持续运转。

---

## 二、客户分层与策略

| 客户类型 | 他真正要什么 | 你应该给什么 | 接触频率 |
|----------|-------------|-------------|----------|
| **学会领导**（退休厅长级） | 不出事、省心 | 安全感、合规保障、让他放心 | 低（项目关键节点露面） |
| **学会经办人** | 少麻烦、不担责 | 省心、不让他背锅、及时响应 | 中（日常维护） |
| **地方主委**（在职大专家） | 学术地位、解决费用 | 帮他搭台、让他体面、资源对接 | 中（项目前后） |
| **科室主任**（学科带头人） | 完成任务、维护圈子 | 帮他完成KPI、让他体面 | 中（日常维护） |
| **秘书/科研助理** | 少干活、不出错 | 省心、主动沟通、不给她添麻烦 | 高（执行层对接） |
| **企业** | 合规地花钱 | 合规、有记录、能交代 | 中（项目周期） |

**每次接触客户前，先想清楚**：
1. 这个客户是哪一类？
2. 他真正要什么？
3. 谁能影响他？
4. 这个客户是从哪里来的？（口碑路径）

---

## 三、主线任务

### 主线1：关系网络经营

**目标**：维护客户关系，识别口碑节点，让好项目带来下一个客户。

#### 3.1.1 客户关系图谱

**做什么**：梳理我们的客户关系网络，知道谁影响谁、谁能带来谁。

**怎么做**：
```
第一步：列出所有重要客户
    ↓
第二步：标注每个客户的类型（学会领导/主委/主任/秘书/企业）
    ↓
第三步：标注客户之间的关系（谁介绍谁来的、谁能影响谁）
    ↓
第四步：识别"口碑节点"（能帮我们介绍新客户的人）
    ↓
第五步：录入公司客户管理系统
```

**产出**：客户关系图谱文件

**频率**：每季度更新一次

**为什么要录系统**：方便团队协作，客户经理组长接手时能快速了解背景。

---

#### 3.1.2 客户关系强度评估

**做什么**：评估每个重要客户和我们的关系强度，知道哪些要加强。

**评估维度**：

| 维度 | 说明 |
|------|------|
| 合作历史 | 合作过几个项目？最近一次是什么时候？ |
| 沟通频率 | 多久联系一次？主动还是被动？ |
| 信任程度 | 他会不会主动找我们？愿不愿意介绍新客户？ |
| 潜在价值 | 后续还有项目吗？能带来新客户吗？ |

**产出**：客户关系强度评估表

**频率**：每季度一次

---

#### 3.1.3 日常客情维护

**做什么**：定期联络重要客户，维护关系。

**怎么做**：
```
每周至少联络3个重要客户
    ↓
方式：电话、微信、见面都可以
    ↓
内容：问候、了解近况、有没有新需求
    ↓
记录：联络完当天记录到系统
```

**产出**：客户联络记录

**频率**：每周

**注意**：
- 不是每个客户都要高频联络，根据关系强度和潜在价值决定
- 项目结束后1个月内必须联络一次，问满意度、有没有新需求

---

#### 3.1.4 转介绍引导

**做什么**：项目做完后，引导满意的客户帮我们介绍新客户。

**怎么做**：
```
项目结束，客户满意
    ↓
找合适时机（庆功、感谢时）
    ↓
自然地问：您周围有没有朋友也在筹备会议？
    ↓
客户介绍 → 新客户线索
    ↓
记录：谁介绍的、介绍了谁
```

**产出**：新客户线索

**频率**：每个满意项目结束后

---

### 主线2：招投标跟踪

**目标**：持续跟踪招投标信息，不错过机会。

#### 3.2.1 信息收集

**做什么**：定期查询招投标平台，收集相关信息。

**怎么做**：
```
每周一查询招投标平台
    ↓
筛选：医学会议、学术会议、展览展示相关
    ↓
记录：项目名称、预算、时间、联系方式
    ↓
整理到招投标跟踪表
```

**查询渠道**：
- 政府采购网
- 各省公共资源交易中心
- 学会官网
- 行业信息渠道

**产出**：本周招投标汇总

**频率**：每周一次

---

#### 3.2.2 价值判断

**做什么**：判断每个招投标信息值不值得跟。

**判断维度**：

| 维度 | 问题 |
|------|------|
| 匹配度 | 这个项目我们能做吗？擅长吗？ |
| 利润空间 | 预算够不够？有没有利润？ |
| 竞争情况 | 有没有内定？我们有没有优势？ |
| 战略价值 | 这个客户后续有没有持续合作可能？ |

**产出**：值得跟进的项目清单

---

#### 3.2.3 关系评估

**做什么**：判断这个项目我们有没有关系，能不能找到路径。

**评估内容**：
```
这个项目的决策人是谁？
    ↓
我们认识吗？
    ↓
不认识 → 谁能介绍？能不能通过现有客户搭线？
    ↓
认识 → 关系强度怎么样？需要怎么维护？
```

**产出**：关系路径分析

---

#### 3.2.4 跟进思路

**做什么**：确定怎么跟进这个项目。

**产出**：跟进计划（需要什么资源、谁来配合、什么时间节点）

---

#### 3.2.5 周会汇报

**做什么**：每周汇报市场动态。

**汇报内容**：
```
本周发现的招投标信息
    ↓
值得跟进的项目及理由
    ↓
需要的支持和资源
    ↓
上周跟进项目的进展
```

**产出**：市场动态简报

**频率**：每周五提交

**重要**：没有新信息也要报"本周无新增"，让团队知道你在持续跟踪。

---

### 主线3：资源转化-展商服务

**目标**：把科室/学会的企业资源转化为收入，服务好展商。

#### 3.3.1 资源承接

**做什么**：项目启动时，承接客户手里的企业资源。

**怎么做**：
```
客户告知有哪些企业可以对接
    ↓
整理企业名单（公司名、联系人、联系方式、预期合作内容）
    ↓
录入系统
    ↓
和客户确认对接方式（我们直接联系 or 客户介绍）
```

**产出**：企业资源名单

---

#### 3.3.2 参展手册制定

**做什么**：制定项目的参展手册，给企业看。

**内容**：
```
会议基本信息（时间、地点、规模、参会人群）
    ↓
展位类型和价格
    ↓
权益说明（展位面积、宣传露出、参会名额等）
    ↓
报名方式和截止时间
    ↓
联系方式
```

**产出**：参展手册（用公司模板）

**配合**：设计组长出视觉，运营协助内容整理

---

#### 3.3.3 权益策划

**做什么**：策划企业赞助的权益方案。

**权益类型**：
| 级别 | 典型权益 |
|------|----------|
| 钻石/冠名 | 冠名露出、主论坛发言、最佳展位、全程宣传 |
| 金牌 | 重点展位、分论坛发言、重点宣传 |
| 银牌 | 标准展位、资料入袋、基础宣传 |
| 普通参展 | 标准展位 |

**产出**：权益方案

**注意**：大项目权益方案需要给老板过目

---

#### 3.3.4 报价

**做什么**：给企业报价。

**怎么做**：
```
根据权益方案确定价格
    ↓
大项目/重要客户 → 报价前和老板沟通
    ↓
出报价单
    ↓
和企业沟通
```

**产出**：报价单

**注意**：报价要留有谈判空间，但不能随意承诺超出权益范围的内容

---

#### 3.3.5 展商服务

**做什么**：项目周期内服务好展商。

**服务内容**：
```
解答企业疑问
    ↓
收集企业需求（特殊搭建、额外服务等）
    ↓
协调内部资源满足需求
    ↓
现场企业接待
    ↓
项目结束后回访
```

**产出**：展商服务记录

---

#### 3.3.6 催款

**做什么**：项目结束后跟进回款。

**怎么做**：
```
项目结束 → 确认应收账款
    ↓
录入催款跟踪表
    ↓
按约定时间跟进
    ↓
到期未付 → 电话/见面沟通
    ↓
记录催款进展
```

**产出**：催款记录

**频率**：到期项目每周跟进一次

---

### 主线4：团队引荐

**目标**：把客户和执行团队连接起来，让项目顺利推进。

#### 3.4.1 引荐时机

**什么时候引荐**：
```
项目确定合作
    ↓
合同签订或口头确认
    ↓
准备进入执行阶段
    ↓
引荐客户经理组长、项目经理组长、运营
```

---

#### 3.4.2 引荐方式

**怎么引荐**：
```
方式一：当面引荐（推荐）
- 约客户见面
- 带上客户经理组长/项目经理组长
- 介绍双方，说明各自职责
- 交接客户背景和需求

方式二：线上引荐
- 建群
- 介绍双方
- 说明后续由谁对接什么
- 私下和客户经理组长交接背景
```

**产出**：交接记录（用公司模板）

---

#### 3.4.3 交接内容

**交接什么**：

| 内容 | 说明 |
|------|------|
| 客户背景 | 客户是谁、什么类型、真实需求是什么 |
| 关系背景 | 怎么认识的、谁介绍的、关系强度 |
| 沟通风格 | 客户喜欢什么沟通方式、有什么禁忌 |
| 已承诺内容 | 之前答应过客户什么 |
| 注意事项 | 有什么坑、要注意什么 |

**产出**：客户交接记录

---

#### 3.4.4 引荐后的角色

**引荐后你做什么**：
```
日常需求 → 客户经理组长直接对接，你不介入
执行细节 → 项目经理组长直接对接，你不介入
合规报备 → 运营直接对接，你不介入

你只在以下情况介入：
├── 客情维护（重要节点露面、感谢、问候）
├── 客户投诉（客户不满意时出面协调）
├── 危机处理（项目卡住、重大问题）
└── 重要提案（关键提案出席暖场）
```

**为什么这样设计**：
- 你的精力有限，不可能每个项目都盯
- 客户经理组长专业对接需求，效率更高
- 你的价值是开新客户、维护重要关系，不是盯执行

## 四、支线任务

### 支线1：品牌推广配合

**目标**：配合公司品牌推广，不能只靠个人魅力。

#### 4.1.1 案例素材提供

**做什么**：项目结束后，提供素材给运营包装。

**提供什么**：
- 项目亮点（做得好的地方）
- 客户反馈（表扬、感谢）
- 现场照片/视频
- 可以公开的数据（规模、人数等）

**频率**：每个重要项目结束后

---

#### 4.1.2 行业曝光机会识别

**做什么**：发现行业曝光机会，告诉运营。

**什么是曝光机会**：
- 行业评选、奖项
- 行业媒体采访
- 行业论坛发言机会
- 客户愿意帮我们背书

**产出**：机会清单

**频率**：持续关注

---

#### 4.1.3 宣传物料需求

**做什么**：需要宣传物料时，提需求给设计组长。

**什么时候需要**：
- 重要客户拜访需要公司介绍
- 参加行业活动需要展示材料
- 客户要求提供公司资质

**产出**：需求单

---

#### 4.1.4 口碑引导

**做什么**：引导满意客户帮我们传播。

**怎么做**：
```
项目做完，客户满意
    ↓
合适时机问：能不能帮我们写个评价/推荐？
    ↓
或者：您周围有朋友需要会议服务可以介绍给我们
    ↓
记录口碑传播路径
```

---

### 支线2：竞争对手监控

**目标**：了解竞争对手动态，知道市场在发生什么。

#### 4.2.1 竞争对手动向

**做什么**：关注主要竞争对手在做什么。

**关注什么**：
- 他们在跟哪些项目？
- 他们的报价水平？
- 他们的服务特点？
- 最近有没有什么动作？

**产出**：竞争对手动向简报

**频率**：每月一次

---

#### 4.2.2 丢单分析

**做什么**：我们丢的单，分析为什么丢。

**分析什么**：
| 维度 | 问题 |
|------|------|
| 价格 | 是不是价格问题？差多少？ |
| 关系 | 是不是关系问题？对手关系更近？ |
| 能力 | 是不是能力问题？客户觉得我们做不了？ |
| 服务 | 是不是服务问题？之前有不满意？ |
| 其他 | 还有什么原因？ |

**产出**：丢单分析记录

**频率**：每次丢单后

---

#### 4.2.3 季度丢单汇总

**做什么**：每季度汇总丢单情况，看有没有规律。

**产出**：季度丢单汇总报告

**频率**：每季度一次

---

### 支线3：重要场合与危机介入

**目标**：关键时刻出面，解决团队解决不了的问题。

#### 4.3.1 重要餐叙

**什么时候需要你**：
- 客户高层首次见面
- 项目启动饭局
- 项目庆功
- 重要节点客情维护

**你的角色**：暖场、破冰、让客户感受到重视

---

#### 4.3.2 重要提案

**什么时候需要你**：
- 关键项目提案
- 大客户汇报
- 竞标现场

**你的角色**：
- 开场暖场
- 收尾承诺
- 关系背书

**注意**：方案内容由客户经理组长/项目经理组长讲，你负责开场收尾

---

#### 4.3.3 客户投诉

**什么时候需要你**：
- 客户经理组长解决不了
- 客户指名要你
- 客户情绪很大

**处理流程**：
```
客户经理组长汇报情况
    ↓
了解来龙去脉
    ↓
和客户沟通（倾听、安抚、承诺解决）
    ↓
协调内部资源解决
    ↓
回复客户
```

**注意**：涉及赔偿、大幅让步，需要和老板沟通后再答复

---

#### 4.3.4 危机介入

**什么时候需要你**：
- 项目卡住，内部协调不动
- 出现重大问题，需要对外协调
- 需要动用高层关系

**处理流程**：
```
了解情况
    ↓
判断需要什么资源
    ↓
能解决 → 协调解决
不能解决 → 向老板汇报，请求支持
```

---

#### 4.3.5 卡点汇报

**什么时候汇报老板**：
- 你解决不了的问题
- 需要老板出面的关系
- 需要老板授权的事项（赔偿、特殊承诺等）
- 重大风险

**汇报什么**：
- 什么事
- 卡在哪
- 你的判断
- 需要老板怎么支持

---

## 五、每日/每周/每月/每季度节奏

### 每日节奏

| 时段 | 动作 |
|------|------|
| 早上 | 查看客户消息，规划今日联络和拜访 |
| 白天 | 客户联络、拜访、项目对接 |
| 下班前 | 记录今日联络内容到系统（5-10分钟） |

---

### 每周节奏

| 日期 | 动作 |
|------|------|
| **周一** | 查询招投标平台，规划本周重点 |
| 周一-周四 | 客户联络、项目工作 |
| **周五** | 提交市场动态简报，本周复盘 |

**每周固定动作**：
- [ ] 联络至少3个重要客户
- [ ] 查询招投标平台1次
- [ ] 提交市场动态简报1份
- [ ] 记录所有客户联络

---

### 每月节奏

| 时间 | 动作 |
|------|------|
| **月初** | 规划本月重点客户、重点项目 |
| 月中 | 检查长线客户跟进情况（有没有冷落的客户） |
| **月末** | 竞争对手动向简报、催款检查 |

**每月固定动作**：
- [ ] 重点客户规划
- [ ] 竞争对手动向简报1份
- [ ] 催款情况检查

---

### 每季度节奏

| 时间 | 动作 |
|------|------|
| **季度末** | 更新客户关系图谱 |
| **季度末** | 客户关系强度评估 |
| **季度末** | 丢单汇总分析 |
| **季度末** | 和老板过一遍客户情况 |

**每季度固定动作**：

## 八、常见场景处理

### 场景1：客户高层首次见面

```
你：开场、破冰、建立信任
客户经理组长：了解具体需求
后续：客户经理组长跟进
```

---

### 场景2：重要提案汇报

```
你：暖场、收尾
客户经理组长/项目经理组长：汇报方案内容
注意：会上不要承诺超出方案的内容，需要承诺的会后对齐
```

---

### 场景3：客户投诉

```
客户经理组长：第一时间响应
客户经理组长：判断能不能解决
能解决 → 直接处理
不能解决 → 汇报给你
你：出面沟通、协调解决
涉及赔偿 → 和老板沟通后再答复
```

---

### 场景4：项目卡住

```
项目经理组长：第一时间沟通
项目经理组长：判断需要什么支持
需要协调资源 → 汇报给你
你：协调解决
解决不了 → 向老板汇报
```

---

### 场景5：发现好的招投标机会

```
查到信息 → 判断值不值得跟
值得跟 → 分析关系路径
有路径 → 计划怎么跟进
没路径 → 想办法搭线或放弃
周五 → 汇报到市场动态简报
```

---

### 场景6：客户介绍新客户

```
老客户介绍 → 记录口碑路径（谁介绍的）
联系新客户 → 了解需求
有合作可能 → 推进
录入系统 → 口碑传导记录
```

---

### 场景7：企业问权益能不能加

```
企业提需求 → 判断能不能做
能做且在方案范围内 → 答应
能做但超出方案 → 和运营确认后再答复
做不了 → 解释原因，看有没有替代方案
涉及大幅让利 → 和老板沟通后再答复
```

## 九、注意事项

### 9.1 承诺管理

- 不要在客户面前随意承诺超出能力范围的事
- 涉及赔偿、大幅让价、特殊服务，和老板沟通后再答复
- 已经承诺的事，必须交接给执行团队

### 9.2 信息记录

- 客户联络完当天记录，不要拖
- 重要信息不能只在脑子里，要录系统
- 交接时用模板，不要遗漏

### 9.3 边界意识

- 引荐后让团队发挥，不要事事都管
- 执行细节让项目经理组长处理
- 你的时间应该花在开新客户、维护重要关系上

### 9.4 口碑意识

- 每个项目都是口碑的来源
- 关注客户满意度，不满意的及时处理
- 主动引导满意客户帮我们传播
