# 成介入场景流程

> 创建日期：2026-01-27
> 所属：组织运作模式 - 补位机制分支
> 状态：待执行验证

---

## 背景

客户经理（如陈霞）在某些场景下需要销售（成）的情绪价值能力补位。但成介入必须有规矩，防止：
1. 刘退出（"有人管了我不管"）
2. 下属依赖（直接找成更舒服）
3. 成变成实质客户经理

---

## 核心原则

| 原则 | 内容 |
|------|------|
| **场景限定** | 成只在特定场景介入，日常不碰 |
| **刘在场** | 成介入时刘必须在场 |
| **责任不转移** | 项目责任始终在刘，成是"资源" |
| **下属不能直接找成** | 需要支持 → 先找刘 → 刘决定是否请成 |

---

## 场景1：客户高层首次见面

| 环节 | 谁做 | 做什么 |
|------|------|--------|
| 开场 | 成 | 介绍公司情况、介绍团队成员 |
| 破冰/寒暄 | 成 | 情绪价值、让客户舒服 |
| 了解需求 | 刘 + 客户经理 | 问具体需求、记录 |
| 后续跟进 | 客户经理 | 整理纪要、推进落地 |
| 关系维护 | 刘把关 | 日常由客户经理，刘兜底 |

---

## 场景2：重要客户的关键汇报/提案

| 环节 | 谁做 | 做什么 |
|------|------|--------|
| 开场 | 成 | 介绍公司情况、介绍成员、暖场 |
| 方案汇报 | 客户经理 | 汇报项目方案/执行计划 |
| 执行说明 | 刘 | 汇报执行保障、资源调配 |
| 客户疑问 | 视问题类型 | 关系类成答、执行类刘/客户经理答 |
| 收尾 | 成 | 总结、表态、情绪收口 |
| 后续跟进 | 客户经理 | 纪要、落地、日常对接 |

---

## 场景3：客户投诉/危机处理

| 环节 | 谁做 | 做什么 |
|------|------|--------|
| 第一时间响应 | 客户经理 | 先稳住、收集情况、报刘 |
| 判断是否需要成 | 刘 | 评估严重程度，决定是否请成 |
| 灭火沟通 | 成 | 出面安抚、承诺解决、情绪价值 |
| 善后方案 | 刘 | 拿出具体解决方案 |
| 执行落地 | 客户经理 | 跟进解决、确认客户满意 |
| 复盘 | 刘 + 客户经理 | 总结问题、避免再犯 |

---

## 场景4：日常对接（成不介入）

| 环节 | 谁做 |
|------|------|
| 日常沟通 | 客户经理 |
| 遇到问题 | 客户经理 → 找刘 |
| 刘处理不了 | 刘 → 判断是否请成 |
| 成不主动介入 | — |

---

## 给各方的说法（不暴露侧写）

| 对象 | 说法 |
|------|------|
| **刘** | "借用销售资源需要你协调，客户高层场合项目负责人必须在场" |
| **成** | "你精力有限，日常的事客户经理处理，你只做关键场合" |
| **客户经理** | "需要销售支持走流程，先跟刘说" |

## 待验证

| 验证什么 | 怎么验证 | 什么时候 |
|----------|----------|----------|
| 成是否守住边界 | 观察成是否接了日常的事 | 下一个项目 |
| 刘是否在场 | 观察成介入时刘有没有退出 | 下一个项目 |
| 客户经理是否绕过刘 | 观察有没有直接找成 | 下一个项目 |
