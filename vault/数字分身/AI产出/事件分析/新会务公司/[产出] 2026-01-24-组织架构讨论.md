# 2026-01-24 组织架构讨论记录

## 背景
讨论新会务公司的组织架构，核心问题：薛和刘分还是合？

---

## 确定的架构：半分组模式

### 核心逻辑
- 不是"刘的项目用刘的人，薛的项目用薛的人"
- 而是：**同一个项目，两边各做各的部分**

| 维度 | 归谁 |
|------|------|
| 项目分配 | 成（按客户喜欢谁，周会确定） |
| 接待侧工作 | 刘的客户经理（公共资源） |
| 执行侧工作 | 薛的团队（公共资源） |
| 现场统筹 | 薛的人 |
| 冲突仲裁 | 老板 |

---

## 人员配置

### 合伙人（6人分30%）

| 合伙人 | 职能 | 出资份额 | 管的人 |
|--------|------|---------|--------|
| 张 | 运营+策划+审计 | 15 | 无（老板给框架，她填充） |
| 成 | 销售+业务分配 | 12 | 无 |
| 刘 | 客户经理统管 | 9 | 客户经理6人 |
| 薛 | 执行统管 | 9 | 女设计PM + 技术执行2人 |
| 杨 | 后勤 | 7.8 | （暂放一边） |
| 周 | 设计 | 7.2 | 设计2人 |

### 分工明细

| 角色 | 负责内容 |
|------|---------|
| 客户经理（刘管6人） | 日程、票务、房间、车辆、注册系统、报账材料、对接秘书 |
| 女设计PM（薛管） | 设计沟通、问平面主K |
| 技术执行（薛管2人） | 搭建、现场执行 |
| 设计（周管2人） | 公共资源，服务所有项目 |
| 成 | 业务分配、主动对接客户防投诉 |
| 张 | 运营、策划（老板给框架）、审计 |

## 项目流程

```
成分配项目（按客户喜欢谁）
     ↓
刘的客户经理：
- 对接秘书（接待秘书+学术秘书）
- 日程汇总分发
- 票务、房间、车辆
     +
薛的团队：
- 女设计PM对接设计需求
- 技术执行做搭建
     ↓
现场：薛的人管
```

## 日程管理

- 原则：谁的项目谁管日程
- 客户经理6人中1人已能胜任
- 客户经理需要单独定义（待后续讨论）
- 接待秘书和学术秘书本质相通，客户经理应该能同时对接

## 张的管理规则

### 性格分析
- 没安全感
- 对刘好（好朋友），对薛打压（威胁+嫉妒）
- 需要老板认可
- 不在意钱，在意归属感和被需要感

### 管理原则
| 原则 | 做法 |
|------|------|
| 给归属感和认可 | 老板多认可她 |
| 不让张直接面对薛 | 薛的报账通过老板协调 |
| 不管"打压" | 张对薛的态度不介入 |
| 让张和刘绑定 | 强化她的安全区 |
| 两个大项目 | 暂时不逼她丢出来 |

## 关于销售设置的讨论

### 行业情况
- 大多数公司没有销售
- 老板自己拿业务

### 老板设置销售的原因
- 为了安全：销售不做事，客户经理接触不到核心
- 风险分散：客户经理走了，客户在销售手上

### 成的真实价值
- 不是"客户关系"，是"主动性"
- 成走了客户流失不了什么，但没人主动找资源
- 开拓了老板以前不理的企业客户战场

### 存在的问题
- 成没有主动引荐老板给客户（老板以前会帮秘书说好话）
- 老板和成互相怀疑
- 短线思维导致气场、沟通、分寸不够

## 待解决问题

1. **客户经理角色定义**：和销售怎么区分？具体职责？
2. **指南制作流程**：客户经理汇总还是单独日程助理？
3. **大项目配人机制**：什么规模配日程助理+接待助理？
4. **成的分配规则细化**：周会怎么操作？
5. **成的客户沉淀机制**：如何让客户关系沉淀到公司而非成个人？

---

## 元准则

> 规则设计基于人性和性格，不是理想化假设

- 对每个人：问"这个人的性格能支持这个职能吗？"
- 不能的话，怎么绕过去？
- 谁和谁不能直接协作？需要什么中间人？

## 成的完整定位

### 成做什么
| 职能 | 具体内容 |
|------|---------|
| 开拓新客户 | 主动找资源、联系潜在客户（尤其企业客户） |
| 业务分配 | 按客户喜欢谁分配，周会汇报 |
| 客户界面 | 让客户记住公司，做客户的第一接触点 |
| 主动对接防投诉 | 项目中主动对接客户，减少投诉风险 |

### 成不做
- 项目统筹（留给刘/薛）
- 具体执行
- 干预刘薛张的工作

### 成的边界
| 边界 | 越界信号 |
|------|---------|
| 分配业务，不是决定业务怎么做 | 干预刘/薛的执行细节 |
| 客户界面，但客户要沉淀到公司 | 客户只认成不认公司 |
| 拿总业绩提成，但要维护老客户 | 只管新客户 |

### 成不是刘薛张的上级
- 分配业务是职能分工，不是管人
- 不决定刘薛张怎么做事
- 不考核刘薛张
- 如果成越界干预，刘薛可以找老板

### 成和刘薛张的配合
```
成分配业务（周会汇报）
        ↓
┌───────┴───────┐
刘（接待侧）    薛（执行侧）
        ↓
      张（运营）
   服务所有项目
```

### 北京公司
- 客户要求的，成想做
- 老板开条件：30%，成当法人
- 一个月成没给方案
- 原则：他要什么就让他承担什么，方案他出不是老板出

### 待正式谈的内容
1. 成的角色和边界
2. 你的期望（引荐客户、沉淀、老客户维护）
3. 他的利益（提成、北京公司）
4. 汇报机制
5. 和刘薛张的配合规则

### 待细化
- 老客户维护具体怎么做
- 成和张的配合（目前没默契）
