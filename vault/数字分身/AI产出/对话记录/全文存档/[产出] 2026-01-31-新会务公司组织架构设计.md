## 组织架构确定

```
老板
 ├── 成（市场拓展）→ 资源承接、展商服务、团队引荐、卡点汇报
 ├── 刘（客户经理）7人 → 客户对接、需求统筹
 ├── 薛（项目经理）+ 陶唐米 → 执行落地
 ├── 张（运营执行）→ 补贴、报备、合规、进度
 └── 周（设计）→ 方向老板定，日常配合刘/薛
```

## 关键决策记录

| 决策 | 内容 |
|------|------|
| 角色命名 | 刘组→客户经理，张→运营执行，成→市场拓展 |
| 团队引荐 | 成引荐刘/薛/张给客户，引荐后各自独立对客户 |
| 招商资源 | 完全依赖科室，结构性约束，老板介入也改变不了 |
| 设计管理 | 大型老板定方向，中小型刘/薛定，周执行 |
| 张的角色 | 降级为运营执行，策划能力在老板身上 |
| 大预算审批 | 张定初稿 → 老板签字 |

## 6项风控机制

| 机制 | 解决的问题 | 执行人 |
|------|-----------|--------|
| 信息回流 | 成退居二线后不知道问题 | 刘/薛/张每周简报 |
| 报价验证 | 报价偏高或偏低 | 大项目老板过目 |
| 客户判断验证 | 成判断失误 | 季度复盘转化率 |
| 长线跟踪提醒 | 成的短线思维忘了跟进 | 张维护表提醒成 |
| 过度承诺约束 | 成答应做不到的 | 重大承诺需老板授权 |
| 催款跟踪 | 成忘了催款 | 张维护表提醒成 |
