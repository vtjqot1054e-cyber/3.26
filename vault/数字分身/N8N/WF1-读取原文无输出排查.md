# WF1「读取原文」无输出 / 像卡住 — 排查

## 现象

- 整表显示「Workflow executed successfully」，但 **读取原文** 右侧 **No output data**。
- INPUT 里 **过滤未处理** 分支提示：**node executed, but no items were sent on this branch**。

## 原因 A（最常见）：上游没有要处理的文件

**读取原文** 只接收 **过滤未处理** 的输出。若 **过滤未处理** 输出 **0 条**（当前所有「对话导出」里的文件都已在待确认/对话删除/对话存档里被排除），则 **读取原文** 不会收到任何 item，**不是卡死，是正常空跑**。

**自检**：在编辑器里只 **Execute step** 运行 **过滤未处理**，看 **OUTPUT** 条数。

- **0 条**：说明当前没有「新导出、且尚未进管线」的文件；要测读取原文，需在 `对话导出/` 放一个**新的**、且文件名对应的 core 名不在待确认/存档里的 `.md`，或先临时移走待确认里同名的待测文件（仅测试用，谨慎）。
- **≥1 条**：再执行 **读取原文**；若仍无输出，查原因 B。

## 原因 B：`vaultPath` 写错（你截图属于这类）

**读取原文 / 写入待确认** 等节点里 `vaultPath` 必须是 **本机 vault 真实路径**，与仓库一致：

- 家里当前约定：`E:\----2\vault\数字分身`

若出现 **F: 盘**、路径里 **问号 `?`**、或 **少写横线**（`----2` 写成 `---`），`fs.readFileSync` 会失败；若此时仍有输入，一般会走 `catch` 返回 `_skip`（但若整段代码被改坏，也可能表现为异常）。

**处理**：在 n8n 里打开对应 Code 节点，将 `vaultPath` 改为与 `vault/数字分身/N8N/01-A对话清洗标注.json` 中一致；或 **重新从该 JSON 导入工作流** 覆盖旧版。

## 原因 C：模式与返回格式

**读取原文** 为 **Run Once for Each Item**，且首行要求 `$json.path` 存在；若上游 item 没有 `path` 字段，会 `return []` 无输出。

仓库中「列出对话导出」会为每条生成 `path: 对话导出/文件名.md`（由 `path.join` 生成，Windows 下为反斜杠，属正常）。

---

**结论**：先确认 **过滤未处理** 是否有输出；再确认 **vaultPath** 无损坏且指向 `E:\----2\vault\数字分身`。
