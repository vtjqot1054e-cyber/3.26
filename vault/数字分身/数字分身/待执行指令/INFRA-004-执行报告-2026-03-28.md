## 四、可能的原因（非技术）

### 1. Cursor未完全重启

**症状**：修改MCP配置后，Cursor仍读取旧配置

**解决**：
- 关闭**所有**Cursor窗口（包括后台进程）
- 使用任务管理器确认没有Cursor进程残留
- 重新启动Cursor

---

### 2. Cursor设置中MCP功能未启用

**检查步骤**：
1. 打开Cursor
2. 按 `Ctrl + ,` 打开设置
3. 搜索 "MCP" 或 "Model Context Protocol"
4. 检查：
   - MCP功能总开关是否开启
   - chroma server是否在列表中
   - chroma server是否被启用（不是灰色/禁用状态）

**截图位置**：
- 设置 → Extensions → MCP
- 或 设置 → Features → Model Context Protocol

---

### 3. 需要查看Cursor MCP Logs

**如何查看**：
1. Cursor底部面板 → 点击 "Output"
2. 右上角下拉菜单 → 选择 "MCP Logs"
3. 查看错误信息

**常见错误**：
- `Failed to start MCP server "chroma"` → 启动失败
- `Command not found` → 命令路径错误（但我们已排除）
- `Connection timeout` → 连接超时
- `Permission denied` → 权限问题

---

### 4. Cursor版本问题

**检查版本**：
- 帮助 → 关于 → 查看Cursor版本号
- MCP功能可能需要特定版本以上

---

### 5. Windows防火墙/杀毒软件

**可能拦截**：
- chroma-mcp.exe 执行
- 本地stdio通信

**解决**：
- 添加 `C:\Users\乐迪\.local\bin\chroma-mcp.exe` 到白名单
- 临时关闭防火墙测试
