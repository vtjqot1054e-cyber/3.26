## 三、MCP工具安装排查（2026-03-28补充）

### 问题发现

用户在新对话窗口中看不到 `chroma_list_collections` 工具，说明MCP服务器未启动。

### 原因分析

1. **uvx命令不存在**：
   ```powershell
   > where.exe uvx
   未找到文件
   ```

2. **uv是Python包，非独立命令**：
   - `uv` 通过 `py -m pip install uv` 安装
   - 作为Python模块存在于 `site-packages/uv/`
   - 没有生成独立的 `uvx.exe` 可执行文件

3. **MCP配置中的command无法执行**：
   - 配置使用 `"command": "uvx"`
   - Cursor启动MCP时找不到该命令
   - 导致MCP服务器启动失败

### 解决步骤

#### 1. 安装chroma-mcp工具

```powershell
py -m uv tool install chroma-mcp
```

**安装详情**：
- 耗时：2分钟18秒
- 依赖包：115个
- 安装位置：`C:\Users\乐迪\.local\bin\chroma-mcp.exe`
- 版本：0.2.6

#### 2. 验证安装

```powershell
> & "C:\Users\乐迪\.local\bin\chroma-mcp.exe" --help
usage: chroma-mcp [-h] [--client-type {http,cloud,persistent,ephemeral}]
                  [--data-dir DATA_DIR] ...
```

#### 3. 更新MCP配置

将 `.cursor/mcp.json` 中的 `command` 改为完整路径：

```json
{
  "mcpServers": {
    "chroma": {
      "type": "stdio",
      "command": "C:\\Users\\乐迪\\.local\\bin\\chroma-mcp.exe",
      "args": [
        "--client-type", "persistent",
        "--data-dir", "E:/家里2/3.26/knowledge_base_v2/vector_store"
      ]
    }
  }
}
```
