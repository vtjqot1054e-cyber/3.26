# SK-08 理论案例匹配

## 触发条件
- 对话中出现用户的行为模式、性格特征、决策风格
- 讨论组织架构、管理问题、人际关系
- 用户要求理论支撑
- **【新增】用户提供外部报告/文章/研究资料**
- **【新增】讨论宏观趋势与用户处境的关系**

---

## 动作

### 1. 识别用户的情况
- 这是什么类型的问题？（个人心理/组织管理/人际关系）
- 核心特征是什么？

### 2. 匹配经典理论
| 领域 | 常用理论 |
|------|---------|
| 个人心理 | 依附理论、大五人格、延迟满足、习得性无助 |
| 组织管理 | 组织承诺、权力基础（French & Raven）、矩阵组织 |
| 人际关系 | 社会交换理论、LMX理论、信任模型 |
| 行为经济学 | 有限理性、损失规避、锚定效应 |
| 领导力 | 情境领导、交易型vs变革型领导 |

### 3. 寻找案例
- 商业案例（企业、创业者）
- 历史案例
- 书籍中的案例

### 4. 输出格式
> 你的情况类似于[理论名称]中描述的[具体概念]。
>
> 案例：[某企业/某人]也遇到过类似情况，他们的做法是...
>
> 这对你意味着...

---

## 【新增】外部知识交叉分析

### 触发条件
- 用户提供外部报告、文章、研究资料
- 用户说"看看这个"、"分析一下这份报告"

### 强制动作

#### 1. 读取用户自述
必须读取 `事件库/我的2026年1月18日报告.txt`（2017-2024创业历程）

#### 2. 交叉分析三问
1. **外部信息说的路线，用户走过没有？结果如何？**
2. **外部信息的建议，与用户人格特质匹配吗？**
3. **矛盾点在哪里？这意味着什么？**

#### 3. 输出格式
```
外部信息说：[路线/建议]
用户历史验证：[你在XX年做过XX，结果是XX]
矛盾点：[具体冲突]
约束性结论：[不是"可以考虑"，而是"这是/不是你的路"]
```

### 错误做法
| ❌ 错误 | ✓ 正确 |
|--------|--------|
| 保存报告，列几个关联点 | 与用户自述深度交叉 |
| "这个报告说的和你业务有关" | "这个报告说的路线，你在2019年走过，结果是被勒索" |
| "可以考虑这个方向" | "这是唯一适合你的路，不是选择" |

## 用户的常见情况与对应理论

| 用户情况 | 对应理论 |
|---------|---------|
| 高宜人性、过度妥协 | Thomas-Kilmann冲突模式（迁就型） |
| 需要合理框架才敢提要求 | 合理化需求、自我效能 |
| 依附强势个体 | 依附理论（焦虑型依附） |
| 面对不确定先放着 | 决策回避、模糊容忍度 |
| 细节先行、框架后置 | 认知风格（场依存vs场独立） |
| 团队成员"只在乎自己" | 委托代理问题、激励相容 |
| 短线思维 | 延迟满足、双曲贴现 |
| 知识垄断 | 权力基础理论（专家权力） |
