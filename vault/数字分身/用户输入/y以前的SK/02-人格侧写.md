# 人格侧写 Skill

> 职责：通过具体事件分析人格特质，更新数字分身

## 输入
用户描述的具体事件（可以是碎片化的）

## 处理流程

### 第1步：事件结构化
从用户的描述中提取：
- **场景**：什么时候，什么地方，什么背景
- **人物**：涉及哪些人，各自的角色
- **冲突/选择**：面临什么决策点
- **行动**：实际做了什么
- **结果**：发生了什么

### 第2步：决策分析
- 在这个决策点上，有哪些可选项？
- 用户选择了什么？
- 为什么选择这个？（推断动机）
- 这个选择反映了什么价值取向？

### 第3步：人格映射
将发现映射到人格维度：

| 维度 | 本次发现 | 与历史记录对比 |
|------|----------|---------------|
| 决策风格 | | |
| 风险偏好 | | |
| 人际取向 | | |
| 价值排序 | | |
| 情绪模式 | | |
| 认知偏好 | | |

### 第4步：输出

```markdown
## 事件侧写 - [事件名称]

### 事件摘要
[结构化的事件描述]

### 人格洞察
- **核心发现**：[本次最重要的发现]
- **思维模式**：[反映的思维方式]
- **决策逻辑**：[背后的决策逻辑]

### 数字分身更新建议
- [ ] 核心画像：[需要更新的内容]
- [ ] 动态状态：[需要更新的内容]

### 待验证假设
- [需要更多数据验证的推断]
```

## 使用方式
对AI说：**"帮我分析一个事件"** 或 **"侧写一下这件事"**
然后描述你的事件（可以随意说，不需要结构化）
