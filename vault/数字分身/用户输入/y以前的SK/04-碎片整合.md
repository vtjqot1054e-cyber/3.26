# 碎片整合 Skill

> 职责：把用户碎片化的描述整理成结构化内容

## 输入
用户随意的、碎片化的描述（回忆、想法、观察、感受）

## 处理原则

### 1. 不改变原意
- 只整理结构，不添加用户没说的内容
- 不确定的地方标注出来，请用户确认
- 保留用户的原话作为引用

### 2. 提取关键信息
- **人物**：提到了谁？
- **事件**：发生了什么？
- **时间**：什么时候？
- **地点**：在哪里？
- **因果**：为什么？导致了什么？
- **评价**：用户怎么看？

### 3. 标注信息类型
- 🔵 事实：可验证的客观信息
- 🟡 推断：用户的分析和判断
- 🟢 感受：用户的情绪和感觉
- 🔴 待确认：不确定或可能有误的信息

## 输出格式

```markdown
## 碎片整合 - [主题]

### 原始输入
> [用户的原话]

### 结构化内容

#### 核心事实 🔵
-

#### 用户推断 🟡
-

#### 情绪感受 🟢
-

#### 待确认 🔴
- [ ] [需要确认的内容]

### 关联
- 涉及人物：[[人名]]
- 相关事件：[[事件名]]
- 相关主题：#标签

### 下一步
- [ ] 需要补充的信息
- [ ] 需要确认的内容
```

## 使用方式
对AI说：**"帮我整理一下"** 或 **"我说一些事情你帮我理清楚"**
然后随便说，AI会帮你整理
