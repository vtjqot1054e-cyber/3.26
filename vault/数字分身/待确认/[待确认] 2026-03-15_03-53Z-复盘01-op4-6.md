## 建议的闭环路径

```
复盘窗口发现问题
     │
     ▼
输出审计报告（给你）
     │
     ▼
你看报告，拍板哪些要改
     │
     ▼ 按修正类型分流
     │
     ├── 规则/SK类（大脑操作纪律、窗口体系、操作手册）
     │   → 你派给架构师改
     │   → 大脑不碰自己的约束规则
     │
     ├── 核心文件类（画像、动态状态）
     │   → 你确认后，复盘窗口自己入库，或派顾问
     │
     └── 指令模板/流程类
         → 你派给架构师改
```

**关键原则：大脑永远不是自己行为修正的执行者。**

- 审计报告 → **你**
- 你拍板 → **架构师**改SK/手册/规则
- 下一个大脑窗口启动 → 自动加载新规则

大脑只是新规则的**接收方**，不是制定方。这样形成的闭环是：复盘发现 → 用户决策 → 架构师执行 → 大脑被动接受。
