N8N 进程没找到。说明 N8N 是在另一个终端窗口里运行的（你手动开的），而那个窗口里**没有设置** `NODE_FUNCTION_ALLOW_BUILTIN`。

**解决方法：**

关掉当前运行 N8N 的终端，然后用以下命令重新启动（在 PowerShell 里运行）：

```powershell
$env:N8N_USER_FOLDER = "D:\n8n\.n8n"
$env:NODE_FUNCTION_ALLOW_BUILTIN = "fs,path,child_process"
Set-Location D:\n8n
npx n8n start
```

或者直接运行：
```powershell
D:\数字分身2\N8N\启动N8N-公司.ps1
```

启动后，N8N 控制台里应该能看到类似 `NODE_FUNCTION_ALLOW_BUILTIN` 的日志，然后再回到 WF1 重新执行节点。

**根本原因**：环境变量必须在启动 N8N **之前**设置，已经在跑的进程不会继承后来改的环境变量。
