# [指令] 工程-SPE双文件去重与WF1修复

> **问题根因**：SpecStory v1.0.0 起，每段 Cursor 对话在 `.specstory/history/` 生成两个文件（已确认 Bug，GitHub issue #196）。
> 旧格式 File A（不带秒，`HH-MMZ`，可读标题，内容更完整）和新格式 File B（带秒，`HH-MM-SSZ`，标题为时间戳，可能截断）。
> **决策：统一保留 File A（旧格式），丢弃 File B。**
> 所有路径以 `E:\数字` 为准。

---

## 任务一：D 盘 SPE 历史文件导入 E 盘（一次性）

### 背景

公司 D 盘 `D:\数字分身2\.specstory\history\` 有历史 SPE 文件（也包含双文件），需要导入到家里 E 盘 `E:\数字\.specstory\history\`，否则两地数据不对齐。

### 执行

写 Node.js 或 Python 脚本：

1. 读取 `D:\数字分身2\.specstory\history\` 下所有 `.md` 文件
2. 去重：按分钟级时间戳（`YYYY-MM-DD_HH-MM`）分组，同组内只保留 File A（不带秒的那个）
3. 将去重后的文件复制到 `E:\数字\.specstory\history\`
4. 如果 E 盘已存在同名文件，跳过不覆盖
5. 输出日志：导入了多少、跳过了多少、丢弃了多少 File B

### 判定 File A vs File B 的规则

```
文件名格式：YYYY-MM-DD_HH-MM-SSZ-title.md  → File B（带秒，丢弃）
文件名格式：YYYY-MM-DD_HH-MMZ-title.md      → File A（不带秒，保留）
```

正则：`/^\d{4}-\d{2}-\d{2}_\d{2}-\d{2}-\d{2}Z/` 匹配到的是 File B。

### 验收

- E 盘 `.specstory/history/` 包含 D 盘所有历史对话（去重后）
- 无 File B 残留
- D 盘原始文件不动（只读操作）

## 任务二：E 盘 SPE 目录存量去重（一次性）

### 背景

任务一导入后，E 盘 `.specstory/history/` 可能仍有本地产生的 File B。

### 执行

1. 扫描 `E:\数字\.specstory\history\` 所有 `.md`
2. 按分钟级时间戳分组
3. 同组内有 File A + File B：删除 File B
4. 同组内只有 File B 没有 File A：**保留 File B**（不能丢数据）
5. 同组内有多个 File A（不同标题，同一分钟）：全部保留（这是不同对话，不是重复）

### 验收

- 每个分钟时间戳组内，不存在 File A 和 File B 共存的情况
