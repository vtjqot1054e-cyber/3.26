## 需要处理的9类过滤项（按复杂度排序）

### 🔴 核心难点：#1 tool-use块删除 + AI回复合并

**问题**：Export Transcript把同一个AI回复中被tool-use分隔的文字段**自动合并**成一段。SpecStory保留了tool-use块，AI文字被拆成多段。

**你必须在SpecStory文件中确认的事**（不能猜）：
- tool-use块的开始标记是什么？（已知线索：`<tool-use data-tool-type=...>`，但你必须在文件中确认）
- tool-use块的结束标记是什么？（`</tool-use>`？还是`</details>`？必须在文件中确认）
- 两个AI文字段之间只隔tool-use时，Export Transcript是如何合并的？（换行连接？直接拼接？必须逐行对比确认）

**合并逻辑**（需要你验证后实现）：
- 如果两个AI文字段之间只隔了tool-use块/thinking块/空行/分隔线（没有User消息），则合并为同一段
- 合并时角色行只保留第一个`_**Agent...**_`
- 合并后的多段文字之间用换行分隔

### #2 thinking块删除

- 格式（已知线索）：`<think><details><summary>Thought Process</summary>...内容...</details></think>`
- 必须在文件中确认实际格式
- 整块删除，含前后的空行和分隔线

### #3 Agent角色行改写

- SpecStory：`_**Agent (model claude-4.6-opus-max, mode Agent)**_`
- Export Transcript：`**Cursor**`
- 注意：不同消息的model和mode可能不同，需要用正则匹配所有变体

### #4 User角色行改写

- SpecStory：`_**User**_` 或 `_**User (时间戳)**_`（两种都有）
- Export Transcript：`**User**`

### #5 SpecStory头部HTML注释删除

- 第1行：`<!-- Generated by SpecStory, Markdown v2.1.0 -->`
- 第3行：`<!-- cursor Session UUID (时间) -->`

### #6 标题行改写

- SpecStory：`# 标题 (时间)`
- Export Transcript：`# 标题` + 换行 + `_Exported on 时间 from Cursor (版本)_`
- 清洗后标题行改为：`# 标题` + 换行 + `_Cleaned from SpecStory on 日期_`

### #7 系统XML标签删除

- 仅4/52个早期文件有
- 标签：`<user_query>`、`<open_and_recently_viewed_files>`、`<system_reminder>`、`<attached_files>`、`<agent_transcripts>`、`<git_status>`、`<rules>`、`<agent_skills>`
- 删除标签**及其内容**之间的所有文本
- ⚠ 注意：`<user_query>`标签内的文本是用户实际发言，**只删标签不删内容**

### #8 tool-use删除后的空分隔线合并

- 删除tool-use后会留下连续的`---`和空行
- 合并为单个`---`加一个空行

### #9 thinking删除后的空分隔线合并

- 同#8逻辑
