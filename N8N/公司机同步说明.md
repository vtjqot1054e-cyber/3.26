# 公司机 N8N 与仓库路径同步

> 杂务维护。仓库根目录下 `N8N\*.json` 与当前 Git 正本一致；**公司**与**家里**仅仓库根路径不同。

## 路径约定

| 环境 | 仓库根 | `vaultPath`（数字分身 vault） |
|------|--------|------------------------------|
| 家里（当前提交默认） | `E:\数字\` | `E:\数字\vault\数字分身` |
| 公司 | `D:\数字分身2\` | `D:\数字分身2\vault\数字分身` |

## 根目录工作流里谁写死了哪边

- `01-A对话清洗标注.json`、`03-用户校对归档.json`：默认 **家里**（`E:\数字\...`），含根目录 `.specstory`：`E:\数字\.specstory\history`。
- `02-用户校对提炼.json`：默认 **公司**（`D:\数字分身2\vault\数字分身`）。

## 公司机 `git pull` 之后（导入 N8N 前）

1. 用编辑器在 **`01-A对话清洗标注.json`**、**`03-用户校对归档.json`** 中全局替换（JSON 转义）：
   - `E:\\数字` → `D:\\数字分身2`
2. 确认 **`02-用户校对提炼.json`** 仍为 `D:\\数字分身2\\vault\\数字分身`（若公司克隆目录名不是 `数字分身2`，再改成你的实际根路径）。
3. Docker 挂载须让容器内能访问上述路径（与现有公司 N8N 卷配置一致）。

## 家里要跑 02 时

将 `02-用户校对提炼.json` 中所有 `D:\\数字分身2` 改为 `E:\\数字` 后再导入（或与 01/03 同批替换规则对称操作）。

详见仓库根 `CLAUDE.md` → **七、路径信息**。
