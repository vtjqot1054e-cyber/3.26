# 访谈总结 SK v1.0

## 一、定位

| 维度 | 说明 |
|------|------|
| **输入** | 已完成二次转写并经用户校准的访谈材料 |
| **输出** | 结构化总结 + 可入库的认知条目 |
| **目的** | 从访谈中提取人、事、物、逻辑，形成可沉淀的认知 |
| **下游** | 人物画像更新、知识库入库、待办跟进 |

---

## 二、核心原则

### 1. 穿透表面看人性

访谈里每个人说的话、做的事，背后都有**行为逻辑**：

| 问什么 | 为什么问 |
|--------|---------|
| 这个人是什么样的人？ | 不是岗位标签，是行为模式 |
| 他在这件事里的立场是什么？ | 利益决定立场 |
| 他为什么这么说/这么做？ | 动机决定行为 |
| 他的逻辑漏洞在哪？ | 认知盲区是改进点 |
| 他和其他人的关系是什么？ | 权力/利益/情感 |

### 2. 识别业务知识

访谈里有大量**具体的规则、数字、流程**，这些是业务运转的底层逻辑：

| 类型 | 示例 |
|------|------|
| **计算规则** | 毛利的10% → 60%给项目组 |
| **扣除项** | 7%税费、20%设计费、20%市场销售 |
| **分配逻辑** | 按比例分红 vs 量化工作价值 |
| **流程** | 开票→扣税→扣代收代付→算毛利→算提成 |
| **行业知识** | 代收代付不能计入提成基数 |

### 3. 提炼管理洞察

访谈里老板（你）的判断、纠偏、教导，往往是**可复用的管理原则**：

| 类型 | 示例 |
|------|------|
| **认知纠偏** | 只看比例不看基数是严重错误 |
| **管理逻辑** | 有些工作不值那么多钱，按比例分不合理 |
| **决策边界** | 账面利润 ≠ 真实利润 |

### 4. 分析事件/冲突

访谈里的争论、分歧、冲突，是**认知碰撞的现场**：

| 问什么 | 为什么问 |
|--------|---------|
| 冲突的表面是什么？ | 现象 |
| 冲突的根因是什么？ | 根因 |
| 双方各自的逻辑是什么？ | 立场 |
| 谁的逻辑有漏洞？ | 盲区 |
| 结果是什么？ | 决定/妥协/搁置 |
| 如果继续下去会怎样？ | 预判 |

### 5. 识别待办/决定

访谈里的承诺、安排、决定，是**需要跟进的事项**：

| 类型 | 示例 |
|------|------|
| **待办** | 董婷去找财务逐笔对账 |
| **决定** | 按7%的比例算提成 |
| **悬而未决** | 分配规则是否调整 |

### 6. 别人不是疯子

当访谈中某人的行为看起来"不合理"时，必须问：

- 他这样做的合理原因是什么？
- 从他的视角看，情况是什么？
- 他可能知道什么我们不知道的？

## 四、入库标准

### 什么值得入库

| 类型 | 入库条件 | 写入位置 |
|------|---------|---------|
| **人物画像** | 新发现的行为模式/动机/盲区 + 有证据 | A系列 |
| **业务知识** | 具体的规则/流程/数字 + 可复用 | B系列 |
| **管理洞察** | 老板的判断/纠偏 + 有因果链 | B系列或M系列 |
| **共识** | 碰撞后双方认可 | R系列 |

### 入库格式（三元结构+出生证明）

每条入库内容必须包含：

```markdown
## [编号] [标题]

### 结论
[一句话核心判断]

### 因果链
- **现象**：[发生了什么]
- **根因**：[为什么发生]
- **改变路径**：[怎么办] 或 [信息不足，待验证]

### 边界
- **适用条件**：[什么情况下成立]
- **不适用条件**：[什么情况下不成立]

### 出生证明
- **来源**：[访谈文件名 + 日期]
- **触发语境**：[访谈中的原文节选]
- **用户确认**：[★共鸣 / ✓确认 / 待验证]
```

## 五、工作流程

```
1. 用户提供已转写校准的访谈文件
      ↓
2. AI 完整阅读，建立全貌
      ↓
3. 按结构提取：
   - 人物画像
   - 业务知识
   - 管理洞察
   - 事件分析
   - 待办/决定
      ↓
4. 生成总结报告
      ↓
5. 标注入库候选条目
      ↓
6. 用户确认/修正
      ↓
7. 按三元结构+出生证明入库
```

## 六、约束

| 约束 | 说明 |
|------|------|
| **不删原文** | 总结是在原文基础上提炼，原文是证据 |
| **不自己打标** | 入库需要用户确认 |
| **不跳过根因** | 每个洞察/事件必须有因果链 |
| **不给模糊建议** | 必须是约束性结论 |
| **别人不是疯子** | 看起来不合理的行为必须追问合理原因 |

```

    ↓

    ↓
【二次转写】← 二次转写 SK
    ↓
用户校准
    ↓

    ↓
入库候选
    ↓
数字真我复盘（碰撞、校准、共鸣）
    ↓
正式入库
```

**访谈总结提供材料，数字真我做最终判断。**
