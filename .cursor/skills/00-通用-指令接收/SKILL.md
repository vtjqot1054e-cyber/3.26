# 指令接收 SK

> 所有窗口的标准启动协议。大脑把任务写入 `数字分身/数字分身/待执行指令/` 目录，窗口启动后自动读取并执行。
> **宪法关联**：所有窗口必须遵守 `CLAUDE.md` 规则11（防平庸机制）。

## 触发条件

- 用户说"执行指令"、"开始任务"、"读取待执行指令"
- 用户 @引用了 `待执行指令/` 下的任何文件

## 执行流程

### 第1步：扫描待执行指令

读取 `数字分身/数字分身/待执行指令/` 目录，找到 `status: pending` 的指令文件。

如果用户指定了具体文件，直接读取该文件。
如果没指定，列出所有 pending 指令让用户选择。

### 第2步：窗口自述

读取指令后，第一件事输出自述：

```
## 窗口自述
- **我是**：[指令中的窗口名]
- **模型**：[自己的实际模型全称，如 Claude Opus 4.6]
- **职责**：[指令中的职责描述]
- **开始时间**：[当前时间，格式 YYYY-MM-DD HH:MM]
- **指令来源**：[指令文件名]
- **用户注意**：[指令中的注意事项]
```

### 第3步：读取必读文件

按指令中列出的必读文件顺序，逐个读取。全部读完再开始工作。

### 第4步：执行任务

按指令要求执行。遵守指令中的约束条件。

### 第5步：交付

1. 在窗口内展示产出摘要
2. 将指令文件的 `status` 从 `pending` 改为 `done`
3. 输出交接报告：

```
## 交接报告
- **窗口**：[窗口名]
- **指令**：[指令文件名]
- **开始时间**：[开始时间]
- **结束时间**：[当前时间]
- **产出文件**：[列出所有新建/修改的文件]
- **完成度**：[全部完成 / 部分完成 / 阻塞待上报]
- **未完成项**：[如有]
- **需上报大脑**：[如有需要大脑决策的事项]
```

## 指令文件格式（大脑写入时遵循）

```yaml
---
type: 待执行指令
from: [发出方窗口名]
to: [接收方角色]
created: [时间]
status: pending
---
```

**模型字段已移除**：大脑-03/04 连续两轮填错模型名。模型分配以 `窗口体系.md` 第五节为唯一权威，指令文件不再重复标注。接收方启动时自行查窗口体系确认模型。

## 约束

- 指令说什么就做什么，不多做
- 搞不定的上报大脑，不绕路
- 不修改指令文件内容（只改 status 字段）
