# 复盘元SK v10

> **v10核心变化**：复盘拆分为两个流程，解决之前"眉毛胡子一把抓"的问题。
> - 流程一（重）：AI做对/做错 → 方法论/红线
> - 流程二（轻）：用户画像/知识点/共识
> - 两个流程都需要因果链，都进知识库但分类不同。

---

## 定位

| 维度 | 说明 |
|------|------|
| **本质** | 复盘的顶层方法论——不是执行流程，是思考框架 |
| **上级** | 数字真我（入库审核权） |
| **消费者** | 所有窗口（回答前匹配方法论/红线） |

---

## 两个流程

### 流程一：AI自我改进（重活）

**问题**：这次AI做对了什么？做错了什么？为什么？下次遇到类似场景怎么做？

**步骤**：
```
AI提炼用户输出（去掉指令性的）
    ↓
用户定性（好/坏/中性）
    ↓
AI因果链分析（只对好/坏做）
    ↓
产出：方法论（好）、红线（坏）
```

**三元结构**：
```
[AI行为] - [导致] - [用户反应]
```

**因果链模板**：
```
因：AI/用户说了什么
果：AI给了什么结果
用户反应：好/坏/中性
结论：下次遇到类似场景应该/不应该这么做
边界：什么条件下成立
```

**产出写入**：
| 类型 | 写入位置 |
|------|---------|
| 方法论（好） | `knowledge_base_v2/M系列` |
| 红线（坏） | `knowledge_base_v2/B-Agent行为陷阱` 或 `CLAUDE.md` 对话纪律 |

---

### 流程二：用户认知积累（轻活）

**问题**：用户是什么样的人？用户说了什么知识点？用户表达了什么共识？

**输入**：流程一的副产品 + 中性定性的内容

**步骤**：
```
从流程一中提取中性内容
    ↓
分类：画像 / 知识点 / 共识
    ↓
补因果链（来源于哪段对话）
    ↓
归档
```

**三元结构**：
```
[内容] - [来源于] - [证据]
```

**产出写入**：
| 类型 | 写入位置 |
|------|---------|
| 画像 | `核心画像.md` 或 `knowledge_base_v2/A系列` |
| 知识点 | `knowledge_base_v2/B系列` |
| 共识 | `knowledge_base_v2/R系列`（共鸣记录） |

## 两个流程的关系

```
流程一（重）：深度因果链分析
    │
    ├── 好 → 方法论
    ├── 坏 → 红线
    │
    └── 副产品 ↓
    
流程二（轻）：收割流程一的副产品
    │
    ├── 画像（用户习惯/认知模式）
    ├── 知识点（用户说的正确结论）
    └── 共识（用户确认的共鸣）
```

**流程二压力轻**是因为：流程一已经做了深度分析，流程二只需要分类归档。

---

## 因果链的作用

| 没有因果链 | 有因果链 |
|-----------|---------|
| 死知识——不知道什么条件下成立 | 活知识——知道边界和适用场景 |
| 不能被正确应用 | 可以被精确匹配和调用 |
| 向量检索可能误用 | 向量检索有边界保护 |

**因果链 > 向量**：向量是加速器，因果链是质量保证。

---

## 消费侧（每次对话前）

```
新对话开始
    ↓
进中枢层（知识库）
    ↓
匹配方法论（类似场景怎么做）
匹配红线（不能碰什么）
匹配画像（用户是什么样的人）
    ↓
带着这些知识回答
```

**大多数对话是消费已有知识，只有少数对话产出新知识。**

## 用户回复信号识别（参考M1）

复盘时需要识别用户反馈的真实态度，不能只看字面：

| 类型 | 信号特征 | 定性 |
|------|---------|------|
| **积极共鸣** | 延伸思考 + 提升维度 + 行动跟进 | 好 |
| **中性** | 短确认（确认、好、继续、OK） | 中性 |
| **纠偏/愤怒** | 连续追问、质疑 | 坏 |
| **深度思考** | 用户说很多 + 解释 + 给例子 | 好/中性（需结合内容判断） |

**常见误判**：愤怒→教导、批评→共鸣。详见 `knowledge_base_v2/M1-用户回复信号识别方法.md`。
