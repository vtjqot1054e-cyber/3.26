# 交接协议 SK

> 每个窗口关闭前必须完成三向交付。不交付就关窗口 = 信息丢失。
> **宪法关联**：所有窗口必须遵守 `CLAUDE.md` 规则11（防平庸机制）。

## 触发条件

- 用户说"关窗口"、"结束"、"交付"、"交接"
- 上下文接近容量极限（AI 应主动提醒）
- 任务全部完成，准备关闭
- 大脑指令"该窗口可以关了"

## 三向交付（强制）

每个窗口关闭前完成三个方向的交付：

### 1. 上级交付（给分大脑/总大脑）

**写在哪**：`数字分身/数字分身/交接备忘.md` 对应区域

**内容**：
- 本次结论（做了什么、产出了什么）
- 状态变化（哪些从"待做"变成"已完成"）
- 需上级决策的事项（如有）

### 2. 续窗口交付（给下一个同名窗口）

**写在哪**：`交接备忘.md` + SpecStory 自动存档

**内容**：
- 断点位置（做到哪了）
- 未完成的待办
- 上下文摘要（下个窗口需要知道的关键信息）

### 3. 关联窗口交付（给其他线的窗口）

**写在哪**：`交接备忘.md` 关联区域

**内容**：
- 影响其他线的发现或结论
- 例：架构师发现"胜率统计全球无现成SK"→ 影响胜率导师，需记录

## 交付模板

```markdown
## 交接报告

**窗口**：[窗口名称]
**角色**：[层级+角色]
**时间**：[开始时间] → [结束时间]

### 上级交付
- **产出文件**：[列出所有新建/修改的文件路径]
- **结论**：[一句话总结本次最重要的结论]
- **状态变更**：[什么从待做变成已完成]
- **需上级决策**：[如有]

### 续窗口交付
- **断点**：[做到哪了]
- **未完成**：[具体待办]
- **上下文**：[下个窗口必须知道的3-5条关键信息]

### 关联窗口
- **影响 [窗口名]**：[什么结论影响了其他线]

### 完成度
[全部完成 / 部分完成 / 阻塞待上报]
```

## 大脑窗口附加：指令完成度核对（强制）

大脑窗口关闭前，除三向交付外，**必须逐条核对本轮指令中的所有任务**：

```markdown
## 指令完成度核对
- P0 [任务名]：✅ / ❌未完成，原因：XX，建议：[延后/取消/转派]
- P1 [任务名]：✅ / ❌未完成，原因：XX
- P2 [任务名]：✅ / ❌未完成，原因：XX
- P3 [任务名]：✅ / ❌未完成，原因：XX
```

防止任何任务被无声遗忘——未完成必须显式标注原因和建议。

## 交付后的文件操作

1. 更新 `交接备忘.md`——将上述内容写入对应区域
2. 更新 `交接备忘.md` 的"当前窗口状态"表——标记本窗口为"已关闭"
3. 如果有待执行指令文件，将 `status` 改为 `done`

## 约束

- **不能只在对话里说**——必须写入文件
- **不能省略空白项**——没有关联窗口影响就写"无"，不能跳过
- **大脑窗口交付更重要**——大脑窗口的交接备忘更新是最高优先级
