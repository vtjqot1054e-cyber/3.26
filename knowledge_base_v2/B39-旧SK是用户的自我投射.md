# 旧SK体系是用户的自我投射

## 这是什么

用户在1月底设计的11个SK（00-10），每一个都指向自己缺失的能力。这不是工具设计，是自我投射。

## 为什么会这样

用户想要什么样的AI，反过来说的就是用户缺什么、怕什么、渴望什么。每个SK背后的痛点比SK本身更有价值。

**因果链**：用户痛点 → 投射为SK设计 → SK反映用户的认知缺口 → 这些缺口是A类画像的原始素材

## 具体对应

- 怕信息丢失 → 长期记忆SK
- 表达碎片化 → 碎片整合SK
- 缺理论功底 → 理论案例匹配SK
- 需要冷静旁观者 → 上帝视角SK

## 对数字真我的意义

理解旧SK不是为了复用架构（架构已完全不同），是为了理解用户。

## 证据

- 用户确认★共鸣："这不是工具设计，这是自我投射"
