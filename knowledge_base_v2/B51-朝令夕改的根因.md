# B51："朝令夕改"的根因

## 结论

用户的"朝令夕改"不是善变，是在用错误的方式（给原则）面对错误的对象（执行者）。

## 三层结构

| 层级 | 原因 |
|------|------|
| **表层现象** | 员工觉得老板经常变 |
| **中层原因** | 用户的思考是连续的，员工的执行是离散的；用户给原则，员工要标准 |
| **深层原因** | 用户期待员工改变思维方式，但大部分员工是执行者不是思考者 |

## 完整循环

```
用户的内部过程：
时刻在想 → 冒出念头 → 觉得必须做 → 讲大道理+举例子说服 → 员工说"懂了"

员工的内部过程：
听老板讲 → 口头说懂 → （实际没想过要改变什么）→ 按自己理解去做

碰撞点：
用户一看结果 → 偏了 → 打断 → 改方向 → 又偏 → 又改
```

## 核心矛盾

| 用户给的 | 员工需要的 |
|---------|-----------|
| 原则、大道理、例子 | 具体标准、明确指令 |
| "你们应该理解为什么" | "告诉我做什么就行" |
| 思考的过程 | 思考的结果 |

## 解法方向

**思考和执行分开**：

```
用户 ←→ 能跟上的人（碰撞/思考层）
              ↓
         转化为具体标准
              ↓
          执行团队
```

## 证据

- 牟成能跟上一部分，因为用户不让他做具体的事
- 用户期望员工思维方式改变、对业务有更深理解
- 用户确认"你的思考速度比员工的执行速度快"这个判断

## 边界

- 需要有人承担"转化层"角色，否则问题不解决
- 转化层的人需要明确预期/角色，避免重蹈B和C的覆辙
