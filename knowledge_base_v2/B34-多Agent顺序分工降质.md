# 多Agent顺序分工的结构性降质

## 这是什么

多Agent在顺序推理任务中性能下降39-70%，错误放大17.2倍。协调开销占总执行时间42%。

## 为什么会这样

管线流水线（标注→信号→提取→复盘→审核）属于典型顺序依赖型任务。每多一次交接，信息衰减一次、上下文丢失一次、理解能力降低一次。分工降质是结构性的，不是实现问题。

**因果链**：每多一次交接 → 信息衰减一次 → 上下文丢失一次 → 理解能力降低一次 → 产出越来越远离用户真实意图

## 什么条件下成立

- 任务是顺序依赖的（后一步需要理解前一步的全部语境）
- 认知类任务（需要理解，不是简单的格式转换）

## 什么条件下不成立

- 任务可并行且互相独立（如投标写标书 vs 技术调研，两件事不互相依赖）
- 纯机械任务（格式转换、文件搬运）

## 怎么改变

顺序依赖型认知任务收归单窗口全栈处理。可并行的独立任务分工有效。

## 证据

- Google Research论文：180种Agent配置实测，顺序推理性能下降39-70%，错误放大17.2倍
- 本系统实证：标注/信号/提取助手从诞生到废弃不到48小时
