# 懂用户→策划→决策→执行→结果→胜率

## 这是什么

这是用户定义的完整价值链。所有窗口的工作最终都要回到这条链路上。

## 为什么重要

数字真我的工作不是"处理待确认文件"，是在这条链路的起点——"懂用户"。没有懂用户，后面的策划、决策都是空中楼阁。没有好的策划和决策（如投标），就无法判断胜负。胜率之我评判的是这条链路的最终结果。

**因果链**：懂用户 → 策划 → 决策 → 执行 → 结果 → 胜率评判

## 什么条件下成立

- 始终成立，这是系统存在的意义

## 证据

- 用户原话："没有好的策划和决策如投标那个部分，又怎么来判断胜负？"
